01Why We Care

This week brings several threads together. We count all possible 4-element poset shapes and understand what makes them distinct. We prove the equivalence between the two definitions of "graded." We introduce the permutahedron — a beautiful geometric object that encodes all permutations of {1,…,n} as vertices, with adjacency given by Kendall tau distance — and see that linear extensions of any poset form a connected subgraph of it. And we meet the 1/3–2/3 conjecture, one of the most famous open problems in combinatorics.

The sum counts the total number of labelled posets on {1,2,3,4}: every labelled poset has exactly one underlying unlabelled shape, so the 16 groups partition all 219 labellings.

How |Aut| is determined
  • Antichain (|Aut|=24): All 4! permutations preserve the empty order relation.
  • Stars (|Aut|=6): The 3 top (or bottom) elements can be permuted freely — 3! = 6.
  • W-shape (|Aut|=4): Can swap the two sources (a↔d) and/or the two targets (b↔c) independently — 2×2 = 4.
  • Diamond (|Aut|=2): Can only swap the two incomparable middle elements.
  • Most asymmetric shapes (|Aut|=1): No non-trivial symmetries → 24 labellings each.

03Graded ↔ Level Partition

Theorem (HW May 1) A finite poset P is graded with height h if and only if it can be partitioned into levels L₁, L₂, …, Lₕ such that:
  1. All minimal elements of P are in L₁.
  2. All maximal elements of P are in Lₕ.
  3. Every cover relation x ⋖ y satisfies x ∈ Lₖ and y ∈ Lₖ₊₁ for some k — i.e., all covers go between consecutive levels.
Proof — Both Directions

(⟹) Graded implies level partition. Suppose P has a rank function rk with rk(x)=0 for minimal elements and rk(y)=rk(x)+1 for all covers x ⋖ y. Define Lₖ = {x ∈ P : rk(x) = k-1} for k = 1,…,h where h = max rk + 1.

  • Minimal elements have rk = 0, so they land in L₁. ✓
  • Maximal elements all have rk = h−1 (since all maximal chains reach rk h−1 by induction on the rank axiom), so they land in Lₕ. ✓
  • Every cover x ⋖ y satisfies rk(y) = rk(x)+1, so x ∈ Lₖ and y ∈ Lₖ₊₁. ✓

(⟸) Level partition implies graded. Suppose P is partitioned into levels L₁,…,Lₕ with the three properties above. Define rk(x) = k-1 for x ∈ Lₖ.

  • Minimal elements are in L₁, so rk = 0. ✓
  • Every cover x ⋖ y has x ∈ Lₖ, y ∈ Lₖ₊₁ (by property 3), so rk(y) = k = rk(x)+1. ✓

This is a valid rank function, so P is graded. ∎

Why minimal elements must ALL be in L₁ (and not just some): If a minimal element m were in Lₖ for k > 1, then there would be no covers going into m from below (m is minimal), but the level structure requires elements in Lₖ to have predecessors in Lₖ₋₁. This means all maximal chains through m would be shorter than chains through elements in L₁, violating the "all maximal chains same length" property.

04Convex Subsets of a Poset

Definition — Convex Subset A subset C ⊆ P is convex (also called order-convex or an interval) if whenever x, y ∈ C and z ∈ P with x ≤ z ≤ y, we have z ∈ C.

The intuition matches the real line: convex subsets of (ℤ, ≤) are intervals like {3,4,5,6}, never {3,5} (the 4 in between escapes).
Examples — Distinguishing Convex from Non-Convex
  • In D₁₂: {2,4,6,12} is convex — every divisor between 2 and 12 that lies above 2 is in the set. (Between 2 and 12: divisors 4 and 6, both included. Between 2 and 4: nothing in D₁₂ strictly between them. ✓)
  • In D₁₂: {2,12} is not convex — missing 4 (since 2|4|12 but 4 ∉ {2,12}).
  • In V-poset a<b, a<c: {b,c} is convex — b and c are incomparable, so no z satisfies b≤z≤c, making the condition vacuously true.
  • In any chain: convex subsets are exactly the contiguous intervals.
Convex Subsets of a Chain of Length n The number of convex subsets of a chain 1 < 2 < ··· < n is:
C(n+1, 2) + 1 = n(n+1)/2 + 1
Why: The non-empty convex subsets are exactly the intervals [i,j] for 1≤i≤j≤n. There are C(n+1,2) = n(n+1)/2 such intervals (choose 2 positions from {1,…,n+1} to mark endpoints). Adding the empty set gives +1.

Example: For chain 1<2<3<4 (n=4): C(5,2)+1 = 10+1 = 11 convex subsets: ∅, {1},{2},{3},{4}, {1,2},{2,3},{3,4}, {1,2,3},{2,3,4}, {1,2,3,4}.
Key Properties
  • Closed under intersection: If C₁ and C₂ are both convex, then C₁ ∩ C₂ is convex. Proof: If x,y ∈ C₁∩C₂ and x≤z≤y, then z ∈ C₁ (C₁ convex) and z ∈ C₂ (C₂ convex), so z ∈ C₁∩C₂. ✓
  • NOT closed under union: In chain 1<2<3, take C₁={1} and C₂={3}. Both are convex (singletons always are). But C₁∪C₂ = {1,3} is not convex — 2 lies between 1 and 3 but is missing.
  • In an antichain: every subset is convex, vacuously (no z can satisfy x≤z≤y when x∥y).

05Polytopes and the Permutahedron

Definition — Polytope A polytope in ℝᵈ is the convex hull of finitely many points (its vertices, in the simplest case). Equivalently, a bounded intersection of finitely many half-spaces.
Definition — Permutahedron Πₙ The permutahedron Πₙ is the convex hull in ℝⁿ of all n! permutations of the vector (1, 2, 3, …, n).

Edge rule Two vertices σ and π are joined by an edge if and only if they differ by exactly one adjacent transposition — swapping two elements in positions i and i+1. This means their Kendall tau distance equals 1. The graph distance on Πₙ equals the Kendall tau distance.

Π₂, Π₃, Π₄

  • Π₂: A single line segment between (1,2) and (2,1).
  • Π₃: A regular hexagon in 2D (after projecting onto the plane x+y+z=6). Six vertices are the 6 permutations of (1,2,3); adjacent vertices differ by one adjacent transposition.
  • Π₄: A truncated octahedron in 3D — 24 vertices, 36 edges, 14 faces (8 hexagonal + 6 square).

Linear Extensions as a Connected Subgraph

Key Theorem — Linear Extensions form a Connected Subgraph If P is a poset on {1,…,n}, the linear extensions of P correspond to a subset of Πₙ's vertices forming a connected subgraph under adjacent-transposition edges.

The reason: if σ is a linear extension and σ(i) and σ(i+1) are incomparable in P, then swapping them gives another linear extension. This is the "swap adjacent incomparable elements" lemma — it says the extension set is closed under certain edges of Πₙ. Any two extensions can be connected by a sequence of such swaps, giving connectedness.

06Order Polytope and Chain Polytope

Definition — Order Polytope O(P) (Stanley 1986) For a poset P on {x₁,…,xₙ}, the order polytope is:
O(P) = {f : P → [0,1] : x ≤ y in P ⟹ f(x) ≤ f(y)}
That is, order-preserving functions from P to [0,1]. It lives in [0,1]ⁿ ⊂ ℝⁿ.
Definition — Chain Polytope C(P) (Stanley 1986) The chain polytope is:
C(P) = {f : P → [0,1] : x < y in P ⟹ f(x) + f(y) ≤ 1}
Functions where comparable elements can't both be large.
Example — Poset {a < b} on two elements

Both polytopes live in [0,1]² with coordinates (f(a), f(b)).

  • O(P): constraint f(a) ≤ f(b). This is the triangle with vertices (0,0), (0,1), (1,1) — the upper triangle of the unit square.
  • C(P): constraint f(a) + f(b) ≤ 1. This is the triangle with vertices (0,0), (1,0), (0,1) — the lower-left triangle.
Order polytope: upper triangle
Chain polytope: lower triangle

Both triangles have area 1/2. Both live in the same unit square but are different objects — O(P) contains the top-right corner (1,1); C(P) contains the bottom-right corner (1,0).

Key Facts
  • The vertices of O(P) are the 0-1 vectors corresponding to upper sets (filters) of P: f(x) = 1 iff x is in the upper set.
  • The vertices of C(P) are the 0-1 vectors corresponding to antichains of P: f(x) = 1 iff x is in the antichain.
  • Both polytopes have the same number of vertices (= number of antichains of P = number of upper sets of P, by the antichain-filter bijection).
  • Both polytopes have the same volume: Vol(O(P)) = Vol(C(P)) = e(P) / n!, where e(P) is the number of linear extensions.

07The 1/3–2/3 Conjecture

Famous Open Problem — Kislitsyn 1968

This is one of the most famous open problems in poset combinatorics, connecting linear extensions to the efficiency of sorting.

Definition — Balance Constant δ(P) For a finite poset P that is not totally ordered, and for two elements x, y, let Pr[x <ₑ y] denote the probability that x comes before y in a uniformly random linear extension e of P.

A pair (x,y) contributes only when x ∥ y (incomparable). Define:
δ(P) = max_{x ≠ y incomparable} min(Pr[x <ₑ y], Pr[y <ₑ x])
How to read it: For each incomparable pair, take the smaller of the two probabilities (always ≤ 1/2, equals 1/2 when perfectly balanced). Then take the maximum over all such pairs. So δ(P) answers: "among all incomparable pairs, how close to 50/50 does the most balanced pair get?"
The 1/3–2/3 Conjecture For every finite poset P that is not a total order:
δ(P) ≥ 1/3
That is, there always exists an incomparable pair (x,y) such that between 1/3 and 2/3 of all linear extensions place x before y.
Current Status
  • Conjectured: Kislitsyn (1968), independently Fredman (1976) and Linial (1984).
  • Best known lower bound: Brightwell–Felsner–Trotter (1995) proved δ(P) ≥ (5−√5)/10 ≈ 0.276.
  • Still open in full generality as of 2026.
  • Known cases: True for posets of width 2 (Linial 1984), and several other special families.
Tight Example — δ(P) = 1/3

Consider the poset P = {a < b} ∪ {c} on three elements: a 2-chain plus one incomparable element.

The three linear extensions are: (a,b,c), (a,c,b), (c,a,b). (a must come before b; c can go anywhere.)

The only incomparable pairs are (a,c) and (b,c):

  • Pr[a <ₑ c] = 2/3 (in 2 of 3 extensions: (a,b,c) and (a,c,b)), Pr[c <ₑ a] = 1/3. So min = 1/3.
  • Pr[b <ₑ c] = 1/3 (only in (a,b,c)), Pr[c <ₑ b] = 2/3. So min = 1/3.

δ(P) = max(1/3, 1/3) = 1/3. This shows the bound 1/3 would be tight if the conjecture is true.

Connection to Sorting If the conjecture holds (δ(P) ≥ 1/3 always), then when sorting elements partially ordered by P, we can always find a comparison that eliminates at least 1/3 of the remaining linear extensions:
Number of comparisons needed ≤ log_{3/2}(e(P)) ≈ 1.71 · log₂(e(P))
This is within a constant factor of the information-theoretic lower bound log₂(e(P)), making the algorithm optimally efficient up to a constant.

08Try It Yourself

Practice Problems — Week 5
Q1List all convex subsets of the antichain on 3 elements {a, b, c}.▼
Solution
Key insight: In an antichain, all elements are incomparable. No z satisfies x ≤ z ≤ y for any distinct x, y (since x ∥ y, neither x ≤ y nor y ≤ x holds). So the condition "x,y ∈ C and x ≤ z ≤ y implies z ∈ C" is vacuously true for every subset.

Every subset of the antichain is convex. There are 2³ = 8 subsets:

∅, {a}, {b}, {c}, {a,b}, {a,c}, {b,c}, {a,b,c}.

All 8 subsets are convex.

Q2List all convex subsets of the V-poset a < b, a < c.▼
Solution
Identify betweenness: The relations are a<b, a<c (and b∥c). What can be "between" two elements?
  • Between a and b: only elements z with a≤z≤b. Since a⋖b (cover), nothing strictly between them.
  • Between a and c: similarly nothing strictly between.
  • Between b and c: b∥c, so the condition is vacuous — nothing can satisfy b≤z≤c.
Since all covers are direct (no intermediate elements) and b∥c, the condition is vacuously satisfied for every subset. All 2³ = 8 subsets are convex.

All 8 subsets are convex: ∅, {a},{b},{c},{a,b},{a,c},{b,c},{a,b,c}.

Q3Find a counterexample showing the union of two convex subsets need not be convex.▼
Solution
Use a chain. In the chain 1<2<3<4, convex subsets are exactly contiguous intervals. Take:
  • C₁ = {1} — convex (single element, trivially).
  • C₂ = {3} — convex (single element, trivially).
  • C₁ ∪ C₂ = {1, 3} — is this convex? Element 2 satisfies 1 ≤ 2 ≤ 3 but 2 ∉ {1,3}.

Counterexample: C₁={1}, C₂={3} in chain 1<2<3. Both convex, but {1,3} is NOT convex.

Q4Prove: the intersection of two convex subsets of P is convex.▼
Solution
Setup: Let C₁, C₂ ⊆ P be convex. Let C = C₁ ∩ C₂. We need to show: if x, y ∈ C and x ≤ z ≤ y, then z ∈ C.
Proof: Suppose x, y ∈ C = C₁ ∩ C₂ and z ∈ P with x ≤ z ≤ y.
Since x, y ∈ C₁ and C₁ is convex: z ∈ C₁.
Since x, y ∈ C₂ and C₂ is convex: z ∈ C₂.
Therefore z ∈ C₁ ∩ C₂ = C. ∎

Proof complete. The key step: z lands in both C₁ and C₂ independently, so it's in their intersection.

Q5How many vertices and edges does Π₅ have? What shape is Π₄?▼
Solution
Π₅: Vertices = 5! = 120. Edges = 5! × (5−1)/2 = 120 × 2 = 240.
Π₄: A truncated octahedron — a 3D solid with 24 vertices, 36 edges, and 14 faces (8 regular hexagons + 6 squares).

Π₅: 120 vertices, 240 edges. Π₄: truncated octahedron (24v, 36e, 14f).

Q6For the diamond poset (a<b, a<c, b<d, c<d with b∥c), locate its 2 linear extensions as vertices of Π₄ and verify they are connected by an edge.▼
Solution
Find extensions: Elements {a,b,c,d} with a first (minimum) and d last (maximum), b and c in either order. Extensions: (a,b,c,d) and (a,c,b,d).
As permutations on {1,2,3,4}: Label a=1,b=2,c=3,d=4. Extensions: (1,2,3,4) and (1,3,2,4).
Compute Kendall tau: The only disagreeing pair is (2,3): in (1,2,3,4) we have 2 before 3; in (1,3,2,4) we have 3 before 2. One disagreement → tau = 1.

Extensions: (1,2,3,4) and (1,3,2,4). tau = 1 → connected by an edge in Π₄. ✓ (b and c are incomparable in the diamond, so swapping them gives another extension.)

Q7Verify that the 16 labelling counts sum to 219 by grouping them.▼
Solution
By number of cover edges:
  • 0 edges: 1 shape → 1 labelling. Subtotal: 1.
  • 1 edge: 1 shape → 12 labellings. Subtotal: 12.
  • 2 edges: 4 shapes → 12+24+12+12 = 60. Subtotal: 60.
  • 3 edges: 8 shapes → 4+24+12+24+24+12+24+4 = 128. Subtotal: 128.
  • 4 edges: 2 shapes → 12+6 = 18. Subtotal: 18.

Total: 1 + 12 + 60 + 128 + 18 = 219 ✓

1 + 12 + 60 + 128 + 18 = 219 ✓

Q8In the 1/3–2/3 conjecture context: for the antichain P = {a, b} (two incomparable elements), what is δ(P)?▼
Solution
Linear extensions of {a∥b}: Both orderings are valid — (a,b) and (b,a). So e(P) = 2.
Compute probabilities: Pr[a <ₑ b] = 1/2 (exactly one of two extensions has a before b). Pr[b <ₑ a] = 1/2.
min(1/2, 1/2) = 1/2. The only incomparable pair (a,b) gives a perfectly balanced 50/50 split.

δ(P) = 1/2 ≥ 1/3. The conjecture holds here (trivially, since this is the most balanced case possible).

δ({a∥b}) = 1/2. The 2-element antichain is perfectly balanced.
