01Why We Care

This week introduces two new tools for thinking about posets. First, we can represent any poset as a graph in two complementary ways — encoding the comparable pairs or the incomparable pairs. Second, and most important for the midterm, we define the dimension of a poset: the minimum number of linear orders whose intersection reconstructs it exactly.

Dimension measures how "complex" a poset is in a precise sense. A total order has dimension 1. An antichain needs dimension 2. And the standard example Sₙ has dimension exactly n, showing dimension can be arbitrarily large.

02CG(P) and IG(P)

Definition — Comparability Graph CG(P) The comparability graph of a poset P has:
  • Vertices: the elements of P.
  • Edges: x ~ y iff x and y are comparable in P — that is, x < y or y < x.
CG(P) is an undirected graph — the direction of the comparison is forgotten.
Definition — Incomparability Graph IG(P) The incomparability graph IG(P) is the complement of CG(P):
  • Vertices: the elements of P.
  • Edges: x ~ y iff x ∥ y (x and y are incomparable in P).
CG(P) and IG(P) together cover every pair of distinct elements exactly once.
CG + IG = complete graph For any two distinct elements x, y: exactly one holds — either x and y are comparable, or they are incomparable. So CG(P) and IG(P) partition the edges of the complete graph Kₙ on n = |P| vertices. Edge in CG ↔ no edge in IG, and vice versa.

Example — D₁₂ = {1, 2, 3, 4, 6, 12}

CG(D₁₂) — comparable pairs
12 edges — all comparable pairs.
The 3 missing edges are 2∥3, 3∥4, 4∥6.
IG(D₁₂) — incomparable pairs
Only 3 edges: 2∥3, 3∥4, 4∥6.
Elements 1 and 12 are isolated
(comparable to everything).
More Examples
  • Chain on n elements: CG = complete graph Kₙ (all pairs comparable). IG = empty graph.
  • Antichain on n elements: CG = empty graph. IG = complete graph Kₙ.
  • 2+2 poset {a<b, c<d}: CG has 2 edges (a~b, c~d). IG has 4 edges (all cross-pairs a~c, a~d, b~c, b~d).
  • Cliques in CG(P) = chains in P. Cliques in IG(P) = antichains in P.

03Intersection of Linear Orders

Definition — Intersection of Linear Orders Given linear orders L₁, L₂, …, Lₖ on the same set, their intersection is the partial order P where:
x < y in P iff x appears before y in EVERY Lᵢ
Written as permutations in one-line notation: x appears to the left of y in all k permutations.
The Key Idea

Each linear order "votes" on the relative order of each pair (x, y). The relation x<y ends up in the intersection only if it wins unanimously — every single Lᵢ places x before y.

If even one Lᵢ disagrees (puts y before x), then x and y are incomparable in the intersection.

Why the intersection is always a valid poset Reflexivity: we use strict < so no self-loops. Antisymmetry: if x before y in all orders, y cannot be before x in all orders simultaneously. Transitivity: if x before y in all, and y before z in all, then x before z in all. ✓

Step-by-step example

Example — Intersection of L₁ = [1,2,3,4] and L₂ = [2,1,4,3]

For each of the C(4,2)=6 pairs, check both:

PairIn L₁?In L₂?In intersection?
1 vs 21 before 2 ✓2 before 1 ✗1 ∥ 2
1 vs 31 before 3 ✓1 before 3 ✓1 < 3
1 vs 41 before 4 ✓1 before 4 ✓1 < 4
2 vs 32 before 3 ✓2 before 3 ✓2 < 3
2 vs 42 before 4 ✓2 before 4 ✓2 < 4
3 vs 43 before 4 ✓4 before 3 ✗3 ∥ 4

Result: {1<3, 1<4, 2<3, 2<4}, with 1∥2 and 3∥4. The poset has two elements {1,2} at the bottom and {3,4} at the top, every bottom below every top, but elements within each level are incomparable.

04Dimension of a Poset

Definition — Dimension dim(P) The dimension of a poset P, written dim(P), is the minimum number of linear extensions of P whose intersection equals P:
dim(P) = min{k : ∃ linear extensions L₁,…,Lₖ with L₁ ∩ ··· ∩ Lₖ = P}
A set {L₁,…,Lₖ} of linear extensions realizing this minimum is called a realizer of P.
Only linear extensions count The Lᵢ must be linear extensions of P — total orders consistent with P (if x<y in P then x before y in every Lᵢ). This guarantees the intersection is always P or coarser than P.

Geometric Intuition

Dimension = number of "axes" needed to represent P

A realizer L₁,…,Lₖ assigns each element x a coordinate vector — its position in each Lᵢ:

x → (position in L₁, position in L₂, …, position in Lₖ)

Then x < y in P iff x's position is smaller than y's in EVERY coordinate.

This embeds P into a product of k chains (a k-dimensional grid). dim(P) = minimum number of axes needed.

  • dim = 1: P is a line — one axis suffices, P is already a total order.
  • dim = 2: P fits in a 2D plane — two axes, like a grid.
  • dim = n: P genuinely needs n-dimensional space.
Geometric example — 2+2 poset in 2D

Realizer for {a<b, c<d}: L₁=[a,b,c,d], L₂=[c,d,a,b].

ElementPos in L₁Pos in L₂2D pointCompared to others
a13(1,3)a<b since (1,3)<(2,4) componentwise ✓
b24(2,4)b∥c since (2,4) vs (3,1): 2<3 but 4>1 ✗
c31(3,1)c<d since (3,1)<(4,2) componentwise ✓
d42(4,2)a∥d since (1,3) vs (4,2): 1<4 but 3>2 ✗

The two chains {a<b} and {c<d} are "oriented perpendicular to each other" in 2D space. That is exactly why they are incomparable — neither is componentwise smaller.

The Two Requirements for a Realizer

What a Realizer Must Do {L₁,…,Lₖ} is a realizer of P iff it satisfies:
  1. (A) Preserve all relations: If x < y in P, then x before y in ALL Lᵢ. (Automatic since each Lᵢ is a linear extension of P.)
  2. (B) Separate all incomparable pairs: If x ∥ y in P, then some Lᵢ has x before y AND some Lⱼ has y before x — so they don't accidentally end up comparable in the intersection.
Condition (B) is the binding constraint. For every incomparable pair, the realizer must "cover" it from both directions.

Dimension of common posets

Standard example SₙnSee below
Every finite poset has a realizer By Szpilrajn's theorem, every partial order can be extended to a linear order. The set of ALL linear extensions of P has intersection equal to P. So dim(P) ≤ e(P) (number of linear extensions). In practice dim(P) is usually very small.

The Standard Example Sₙ

Definition — Standard Example Sₙ Sₙ has 2n elements {a₁,…,aₙ, b₁,…,bₙ} with the single rule:
aᵢ < bⱼ iff i ≠ j
So each aᵢ is below every bⱼ except bᵢ. Each diagonal pair (aᵢ, bᵢ) is incomparable.

dim(Sₙ) = n.
S₂ is the 2+2 poset

S₂ = {a₁,a₂,b₁,b₂} with a₁<b₂ and a₂<b₁. Incomparable: a₁∥b₁, a₂∥b₂.

Relabel a₁→a, a₂→c, b₁→d, b₂→b: we get {a<b, c<d} — exactly the 2+2 poset.

dim(S₂) = 2. Realizer: L₁=[a₁,b₂,a₂,b₁], L₂=[a₂,b₁,a₁,b₂]. ✓

S₃ — 6 elements, dim = 3

S₃ = {a₁,a₂,a₃,b₁,b₂,b₃}. Relations: a₁<b₂,b₃; a₂<b₁,b₃; a₃<b₁,b₂. Incomparable: (a₁,b₁), (a₂,b₂), (a₃,b₃) and all aᵢ∥aⱼ, bᵢ∥bⱼ.

The 3 diagonal pairs each need separate linear extensions → dim ≥ 3. And an explicit 3-element realizer exists → dim = 3.

Why dim(Sₙ) = n — the key argument

Lower bound (dim ≥ n): Consider the diagonal pair (aᵢ, bᵢ) — they are incomparable, so the realizer must have some Lₖ with bᵢ before aᵢ. But since aⱼ<bᵢ for all j≠i, every linear extension must have aⱼ before bᵢ. So in Lₖ: all a₁,…,aᵢ₋₁,aᵢ₊₁,…,aₙ come before bᵢ, then bᵢ, then aᵢ. This extension is "dedicated" to separating pair (aᵢ,bᵢ). Since the n diagonal pairs are independent, you need at least n such dedicated extensions.

Upper bound (dim ≤ n): An explicit n-element realizer can be constructed.

05Worked Examples

🟢 Example 1 — Compute CG and IG of the 2+2 poset

Poset: {a<b, c<d}, elements {a,b,c,d}.

Comparable pairs: (a,b) and (c,d). CG has exactly 2 edges: a~b and c~d.

Incomparable pairs: All 4 cross-pairs between {a,b} and {c,d}: a~c, a~d, b~c, b~d. IG has 4 edges.

Check: 2+4 = 6 = C(4,2). ✓ CG + IG = complete graph K₄.

🟡 Example 2 — Build an intersection from scratch

Given: L₁=[a,b,c,d], L₂=[b,a,d,c].

PairL₁L₂Intersection
{a,b}a before bb before aa ∥ b
{a,c}a before ca before ca < c
{a,d}a before da before da < d
{b,c}b before cb before cb < c
{b,d}b before db before db < d
{c,d}c before dd before cc ∥ d

Result: {a<c, a<d, b<c, b<d} with a∥b and c∥d. This is the same "2-below-2" structure as the first intersection example.

🔴 Example 3 — Verify a realizer for the diamond

Poset: Diamond {a<b, a<c, b<d, c<d} with b∥c.

Proposed realizer: L₁=[a,b,c,d], L₂=[a,c,b,d].

Check (A) — valid linear extensions?

  • a<b: L₁ pos 1<2 ✓, L₂ pos 1<3 ✓
  • a<c: L₁ pos 1<3 ✓, L₂ pos 1<2 ✓
  • b<d: L₁ pos 2<4 ✓, L₂ pos 3<4 ✓
  • c<d: L₁ pos 3<4 ✓, L₂ pos 2<4 ✓

Check (B) — incomparable pair (b,c) separated?

  • L₁: b at pos 2, c at pos 3 → b before c.
  • L₂: c at pos 2, b at pos 3 → c before b.

L₁ and L₂ disagree on (b,c) → b∥c in intersection. ✓

Valid realizer. dim(diamond) = 2.

06Try It Yourself

Practice Problems — Comparability Graphs, Intersection, Dimension
Q1For the chain 1<2<3<4, describe CG(P) and IG(P) completely.▼
Solution
CG(P): All pairs in a chain are comparable. CG = complete graph K₄ with C(4,2) = 6 edges.
IG(P): No pairs are incomparable. IG = empty graph, 0 edges.

CG = K₄ (6 edges). IG = empty (0 edges).

Q2Compute the intersection of L₁=[1,3,2,4] and L₂=[3,1,4,2]. Which pairs end up incomparable?▼
Solution
Check each pair — does x appear before y in BOTH?
  • (1,2): L₁ pos 1<3 ✓; L₂ pos 2<4 ✓ → 1<2
  • (1,3): L₁ pos 1<2 ✓; L₂ pos 2>1 ✗ (3 is at pos 1, before 1) → 1∥3
  • (1,4): L₁ pos 1<4 ✓; L₂ pos 2<3 ✓ → 1<4
  • (2,3): L₁ pos 3>2 ✗ (3 before 2 in L₁) → 2∥3
  • (2,4): L₁ pos 3<4 ✓; L₂ pos 4>3 ✗ (4 before 2 in L₂) → 2∥4
  • (3,4): L₁ pos 2<4 ✓; L₂ pos 1<3 ✓ → 3<4

Relations: {1<2, 1<4, 3<4}. Incomparable: 1∥3, 2∥3, 2∥4.

Intersection = {1<2, 1<4, 3<4}. This is a skew/N-like poset.

Q3Show dim(antichain on 4 elements) = 2 by giving an explicit realizer.▼
Solution
dim ≥ 2: There are incomparable pairs (e.g. a∥b), so at least 2 extensions are needed.
Explicit realizer: L₁=[a,b,c,d] and L₂=[d,c,b,a] (the reverse). For any pair (x,y): in L₁ x comes before y, in L₂ y comes before x. So no pair satisfies "x before y in BOTH" — the intersection is the empty poset = antichain. ✓

dim = 2. Realizer: L₁=[a,b,c,d], L₂=[d,c,b,a].

Q4Why does a total order (chain) have dim = 1?▼
Solution
A total order is already a linear extension of itself. The intersection of {L} where L is the chain is just L — which equals the chain P. So one linear extension suffices.
Cannot do it with 0 extensions (the minimum is always ≥ 1). And since all pairs are comparable (no incomparable pairs to separate), condition (B) is vacuously satisfied with a single extension.

dim(chain) = 1. The chain is its own realizer.

Q5Verify that L₁=[a,b,c], L₂=[a,c,b] is a valid realizer for the V-poset {a<b, a<c} with b∥c.▼
Solution
Both are linear extensions? a before b: L₁ pos 1<2 ✓, L₂ pos 1<3 ✓. a before c: L₁ pos 1<3 ✓, L₂ pos 1<2 ✓. ✓
Incomparable pair (b,c) separated? L₁ has b before c (pos 2<3). L₂ has c before b (pos 2<3). Disagree → b∥c in intersection. ✓
Intersection = V-poset? a<b ✓, a<c ✓, b∥c ✓.

Valid realizer. dim(V-poset) = 2.

Q6Identify the standard example S₂. What familiar poset is it? What is its dimension?▼
Solution
S₂ has elements {a₁,a₂,b₁,b₂} with a₁<b₂ and a₂<b₁. Incomparable: a₁∥b₁, a₂∥b₂, a₁∥a₂, b₁∥b₂.
Identification: Relabel a₁→a, a₂→c, b₁→d, b₂→b: get {a<b, c<d} = the 2+2 poset.
dim(S₂) = 2.

S₂ = 2+2 poset. dim = 2.

Q7Explain in your own words: why does dim(Sₙ) ≥ n? What forces you to need at least n linear extensions?▼
Solution

The n diagonal pairs (a₁,b₁),…,(aₙ,bₙ) are all incomparable. Focus on pair (aᵢ,bᵢ):

Since aᵢ∥bᵢ, the realizer needs some Lₖ with bᵢ before aᵢ.
But since aⱼ<bᵢ for all j≠i, every linear extension must place aⱼ before bᵢ. So in Lₖ the order is: (all aⱼ, j≠i) …then bᵢ …then aᵢ. This extension is "dedicated" to separating (aᵢ,bᵢ).
Each of the n pairs needs its own dedicated extension, and these requirements are incompatible to combine. So ≥ n extensions are required.

dim(Sₙ) ≥ n because each diagonal pair (aᵢ,bᵢ) forces a dedicated linear extension, and the n requirements are independent.

Q8Compute the intersection of σ=[2,3,1,4] and τ=[1,2,3,4]. Give the resulting poset as a list of relations.▼
Solution
Positions in σ=[2,3,1,4]: element 2 at pos 1, element 3 at pos 2, element 1 at pos 3, element 4 at pos 4.
Positions in τ=[1,2,3,4]: element 1 at pos 1, element 2 at pos 2, element 3 at pos 3, element 4 at pos 4.
Check each pair:
(1,2): σ has 2 before 1 ✗ → 1∥2
(1,3): σ has 3 before 1 ✗ → 1∥3
(1,4): σ: 1 at pos 3, 4 at pos 4 ✓; τ: 1 at pos 1, 4 at pos 4 ✓ → 1<4
(2,3): σ: 2 at pos 1, 3 at pos 2 ✓; τ: 2 at pos 2, 3 at pos 3 ✓ → 2<3
(2,4): σ: 2 at pos 1, 4 at pos 4 ✓; τ: 2 at pos 2, 4 at pos 4 ✓ → 2<4
(3,4): σ: 3 at pos 2, 4 at pos 4 ✓; τ: 3 at pos 3, 4 at pos 4 ✓ → 3<4

Note: 2<4 is implied by 2<3<4, so the Hasse diagram has: chain 2<3<4, and element 1 which only satisfies 1<4 directly. Elements 1 and 2 are incomparable; 1 and 3 are incomparable.

Intersection: {1<4, 2<3, 2<4, 3<4}. Hasse: chain 2<3<4, with 1 only below 4.

Key definitions to recall:

  • CG(P): x~y iff x<y or y<x in P.
  • IG(P): x~y iff x∥y in P. (Complement of CG(P).)

Connections:

  • Cliques in CG(P) = chains in P.
  • Cliques in IG(P) = antichains in P.
