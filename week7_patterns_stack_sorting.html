02dim(Sₙ) = n — The Complete Proof

Recall the standard example Sₙ from Week 6: elements {a₁,…,aₙ, b₁,…,bₙ} with aᵢ < bⱼ iff i ≠ j. The diagonal pairs (aᵢ, bᵢ) are all incomparable. We showed dim(Sₙ) ≤ n in Week 6 by exhibiting n linear extensions. Now we prove dim(Sₙ) ≥ n.

Theorem (HW May 11) — dim(Sₙ) = n exactly The standard example Sₙ has dimension exactly n.
Proof of lower bound — dim(Sₙ) ≥ n

Key lemma: in any linear extension of Sₙ, at most one diagonal pair (aᵢ, bᵢ) can have bᵢ before aᵢ.

Proof of lemma: Suppose for contradiction that some linear extension L has both bᵢ before aᵢ AND bⱼ before aⱼ for some i ≠ j. We derive a cycle:

  • aⱼ < bᵢ in Sₙ (since j ≠ i) → aⱼ before bᵢ in L
  • bᵢ before aᵢ in L (assumption)
  • aᵢ < bⱼ in Sₙ (since i ≠ j) → aᵢ before bⱼ in L
  • bⱼ before aⱼ in L (assumption)

Chaining these: aⱼ → bᵢ → aᵢ → bⱼ → aⱼ — a cycle in L. But L is a total order, which has no cycles. Contradiction. ✗

Conclusion: Since each linear extension can "separate" at most one diagonal pair (aᵢ, bᵢ), and we have n incomparable diagonal pairs that each must be separated, the realizer needs at least n extensions. Therefore dim(Sₙ) ≥ n.

Combined with dim(Sₙ) ≤ n (the explicit realizer from Week 6): dim(Sₙ) = n. ∎

Concrete example — S₂

S₂ = {a₁,a₂,b₁,b₂} with a₁<b₂ and a₂<b₁. Diagonal pairs: (a₁,b₁) and (a₂,b₂).

If some extension L has b₁ before a₁: then a₂ before b₁ (since a₂<b₁) and a₁ before b₂ (since a₁<b₂). So L contains: a₂→b₁→a₁→b₂. Can L also have b₂ before a₂? That would give b₂→a₂→b₁→a₁→b₂ — cycle! So L cannot also separate (a₂,b₂). Two separate extensions needed. dim(S₂) = 2. ✓

03Pattern Avoidance

Definition — Pattern Containment and Avoidance A permutation σ ∈ Sₙ contains the pattern π ∈ Sₖ (k ≤ n) if there exist indices i₁ < i₂ < ··· < iₖ such that the relative order of σ(i₁), σ(i₂), …, σ(iₖ) is the same as the relative order of π(1), π(2), …, π(k).

σ avoids π if it does not contain π. We write Avₙ(π) for the set of π-avoiding permutations of length n.
Examples
  • (3,1,4,2) contains 231: take positions 1,2,4 with values (3,1,2) — relative order is 2,1,3? No. Take positions 1,3,4 with values (3,4,2) — relative order 2,3,1 = 231. ✓
  • (1,3,2,4) avoids 321: no three positions with values in strictly decreasing order. ✓
  • (4,3,2,1) contains every pattern in S₃ (it's the reverse permutation).

321-Avoiding Permutations

Characterization of 321-Avoiding Permutations The following are equivalent for a permutation σ:
  1. σ avoids the pattern 321.
  2. σ has no decreasing subsequence of length 3 (longest decreasing subsequence has length ≤ 2).
  3. σ can be partitioned into at most 2 increasing subsequences.
Why (1) ↔ (2): A 321 pattern IS a decreasing subsequence of length 3. ✓
Why (2) ↔ (3): By Dilworth's theorem, the longest decreasing subsequence has length ≤ 2 iff the permutation (viewed as a poset under dominance) has width ≤ 2, iff it can be partitioned into at most 2 chains = 2 increasing subsequences. ✓
Examples — splitting into 2 increasing subsequences
  • (3,1,4,2): Split as {3,4} and {1,2} — both increasing. No 321 pattern. ✓
  • (2,4,1,3): Split as {2,4} and {1,3}. ✓
  • (3,2,1,4): Contains 3>2>1 — a 321 pattern. Cannot split into 2 increasing. ✗
Connection to Week 4 — Erdős–Szekeres By Erdős–Szekeres, any permutation of length n²+1 contains either an increasing subsequence of length n+1 or a decreasing subsequence of length n+1. A 321-avoiding permutation has longest decreasing subsequence ≤ 2. If it's 321-avoiding of length n, the longest increasing subsequence is at least ⌈n/2⌉ (by Erdős–Szekeres with r=2, s=⌈n/2⌉).

Permutation Classes

Definition — Permutation Class A permutation class is a set C of permutations that is closed downward under pattern containment: if σ ∈ C and π is a pattern of σ (π is contained in σ), then π ∈ C.

Every permutation class is of the form Av(F) = {σ : σ avoids all π ∈ F} for some set F of "forbidden patterns."
Examples
  • Av(321) = all 321-avoiding permutations (closed: any pattern of a 321-avoider avoids 321 too).
  • Av(231) = all 231-avoiding permutations = stack-sortable permutations.
  • Av(12) = only the decreasing permutations (n, n−1, …, 1) for each n.
  • Av(∅) = all permutations (no forbidden pattern).

04Stack Sorting and 231-Avoiding Permutations

Definition — Stack-Sort Algorithm Given input permutation σ = (σ(1), σ(2), …, σ(n)), process left to right with a stack:
  1. If the stack is empty or the stack top is greater than the current input element: PUSH the input element onto the stack.
  2. Otherwise (stack top ≤ current input): POP the stack top to the output, and repeat the comparison.
  3. After all input is processed: pop remaining stack elements to output.
The algorithm succeeds if the output is the sorted sequence 1, 2, …, n.
Trace example — σ = (2, 3, 1)
Input 2:PUSH 2stack=[2]output=[]
Input 3:top=2 < 3 → POP 2stack=[]output=[2]
PUSH 3stack=[3]output=[2]
Input 1:top=3 > 1 → PUSH 1stack=[3,1]output=[2]
Done:POP 1, POP 3stack=[]output=[2,1,3]

Output [2,1,3] ≠ [1,2,3]. Not sorted! Note: (2,3,1) contains the 231 pattern (values 2<3 with 1<2 after).

Theorem (Knuth 1968) — Stack-Sortable ↔ 231-Avoiding A permutation σ is stack-sortable if and only if σ avoids the pattern 231.

Therefore: |stack-sortable perms of length n| = |Avₙ(231)| = Cₙ.

Proof: The c,k,t Argument

Proof — (⟹) Not stack-sortable implies contains 231

Suppose stack-sort fails on σ — the output is not sorted. We show σ contains a 231 pattern.

Setup the triple (t, c, k). Since the output is not sorted, at some point an element is output before a smaller element that should have come first. Trace back to identify:
  • t = an element that has been pushed onto the stack and is waiting
  • c = the element that caused t to be popped (c > t, so c triggers popping t out of order)
  • k = an element that comes AFTER c in the input with k < t

In σ, the relative positions are: t appears before c, which appears before k.

The values satisfy c > t > k.
  • c > t: because c arriving caused t to be popped (the algorithm pops when top < input).
  • t > k: because k < t means k needs to come before t in sorted output, but t was already output — that's the failure.
Positions of (t,c,k) in σ form a 231 pattern. The element t appears at some position i, c at position j > i, and k at position l > j. Values: t < c (since c > t) and k < t (since t > k). So the relative order is: t=medium, c=large, k=small — exactly the pattern 231. ✓

Therefore σ contains 231. ∎

Proof — (⟸) 231-Avoiding implies stack-sortable

By strong induction on n. Base case n=1: trivially sorted.

Inductive step: let n be the largest element of σ. Say n is at position k. Run stack-sort up to but not including n.

Key observation: since σ avoids 231, the elements to the left of n (positions 1..k-1) must all be less than the elements to the right of n (positions k+1..n). (If some left value a > some right value b, then with n in the middle: since a < n, this creates... actually the argument uses the 231-avoidance more carefully on the subsequences.)

The induction applies to both the left subsequence (positions 1..k-1) and right subsequence (positions k+1..n) separately — both are 231-avoiding permutations of smaller length. Stack-sort correctly sorts each part, and n (being the maximum) can always be placed last. ∎

Stack-sort bijection → Dyck paths

For every 231-avoiding σ, run stack-sort and record: each PUSH = U step, each POP = D step. This gives a path with n U's and n D's that never goes below 0 (the stack never empties before we're done). This is a Dyck path, and the map σ ↦ path is a bijection.

(1,3,2)push 1, pop 1, push 3, push 2, pop 2, pop 3UDUUDD

All 5 = C₃ Dyck paths of length 6 appear exactly once. ✓

05The 321-Avoiding Machine (HW May 15)

The 2-Queue Machine (Patience Sort into 2 Increasing Piles) Process elements of σ left to right, maintaining two queues (piles) Q₁ and Q₂:
  1. For each element x: if Q₁ is empty OR the last element of Q₁ is less than x, append x to Q₁.
  2. Else if Q₂ is empty OR the last element of Q₂ is less than x, append x to Q₂.
  3. Else: FAIL (σ contains a 321 pattern).
If it never fails: merge the two queues in sorted order to get output 1,2,…,n.
Theorem — The machine sorts iff σ avoids 321 The 2-queue machine succeeds (never fails) on σ if and only if σ avoids the pattern 321.

Proof sketch: If σ contains 321 (values a > b > c at positions i<j<l), then when processing c, both queues have top ≥ c (since b and something ≥ b are already placed), so both queues refuse c → FAIL. Conversely, if σ is 321-avoiding, it can be partitioned into 2 increasing subsequences (by the characterization above), so 2 queues always suffice.
Counting 321-Avoiding Permutations
|Avₙ(321)| = Cₙ = C(2n,n) / (n+1)
Proof: All six patterns of length 3 give Catalan-many avoiding permutations (Simion–Schmidt 1985). In particular |Avₙ(321)| = |Avₙ(231)| = Cₙ. The cleanest proof uses the Catalan recurrence directly:

Let f(n) = |Avₙ(231)|. Place the maximum element n at position k. For 231-avoiding permutations, the k−1 elements to the left of n must be exactly {1,…,k−1}, and the n−k elements to the right must be exactly {k,…,n−1}. Each side is independently 231-avoiding. So:

f(n) = ∑_{k=1}^{n} f(k−1) · f(n−k) with f(0)=f(1)=1
This is the Catalan recurrence, giving f(n) = Cₙ. ✓

06Dyck Paths and Catalan Numbers

Definition — Dyck Path A Dyck path of length 2n is a lattice path from (0,0) to (2n,0) using:
  • Up steps U = (+1,+1)
  • Down steps D = (+1,−1)
that never goes below the x-axis (height ≥ 0 at all times).

Equivalently: a sequence of n U's and n D's such that every prefix has at least as many U's as D's.
All Dyck paths for n = 3 (length 6)

There are C₃ = 5 Dyck paths. Heights at each step:

Catalan Number Formula The number of Dyck paths of length 2n equals the nth Catalan number:
Cₙ = C(2n, n) / (n+1) = 1/(n+1) · C(2n,n)

Values: C₀=1, C₁=1, C₂=2, C₃=5, C₄=14, C₅=42, C₆=132, C₇=429.

Bertrand's Ballot Theorem and the Cycle Lemma

Bertrand's Ballot Theorem (1887) In an election where candidate A receives p votes and candidate B receives q votes, with p > q, the probability that A is strictly ahead of B throughout the entire count (assuming votes are counted in a uniformly random order) is:
P(A always strictly ahead) = (p − q) / (p + q)
Proof — The Cycle Lemma

Represent each vote for A as +1 and each vote for B as −1. A counting order is a sequence of p values (+1) and q values (−1).

Arrange the votes in a circle. Consider all p+q positions in the circular arrangement as possible "starting points." We claim: exactly p−q of the p+q starting positions give a sequence where A is always strictly ahead (all partial sums positive).
Prove the claim. For each starting position, define the "running total" as we go around the circle. Two different starting positions cannot both give "always positive" sequences from the same circular arrangement, UNLESS the total sum is positive (p > q ensures the sum is p−q > 0). A careful argument shows that among all p+q rotations of any fixed circular arrangement, exactly p−q start "winning" sequences. (This is the Dvoretzky–Motzkin cycle lemma.)
Count. Total sequences = C(p+q, p). Winning sequences = C(p+q,p)·(p−q)/(p+q). Probability = (p−q)/(p+q). ✓
Deriving Cₙ = C(2n,n)/(n+1) from the Ballot Theorem

A Dyck path of length 2n is a sequence of n U's (+1) and n D's (−1) with all partial sums ≥ 0.

Relate to ballot problem. Consider sequences of n+1 U's and n D's (total 2n+1 steps, sum = 1). By the ballot theorem with p=n+1 and q=n: P(always strictly ahead) = (n+1−n)/(n+1+n) = 1/(2n+1).
Count. Number of such "always positive" sequences = C(2n+1, n+1)·1/(2n+1) = C(2n+1, n+1)/(2n+1).
Simplify.
C(2n+1, n+1) / (2n+1) = (2n+1)!/((n+1)!·n!) / (2n+1) = (2n)!/((n+1)!·n!) = C(2n,n)/(n+1) = Cₙ
Connect to Dyck paths. Removing the last U from each "always positive" (n+1 U, n D) sequence gives a bijection to sequences of n U's and n D's with all partial sums ≥ 0 — precisely Dyck paths of length 2n. So the number of Dyck paths = Cₙ. ✓

Equivalently: P(random path ending at 0 is a Dyck path) = Cₙ / C(2n,n) = 1/(n+1).

The Big Picture — What Equals Cₙ All of the following count Cₙ for each n:
  • Dyck paths of length 2n
  • 231-avoiding permutations of {1,…,n}
  • 321-avoiding permutations of {1,…,n}
  • Stack-sortable permutations of {1,…,n}
  • Ways to triangulate a convex (n+2)-gon
  • Full binary trees with n+1 leaves
  • Valid parenthesizations of n+1 factors

07Stanley–Wilf Conjecture (Now a Theorem)

Marcus–Tardos Theorem (2004) For every permutation pattern σ, there exists a finite constant c(σ) such that:
|Avₙ(σ)| ≤ c(σ)ⁿ for all n
Equivalently, the Stanley–Wilf limit L(σ) = lim_{n→∞} |Avₙ(σ)|^{1/n} exists and is finite.
What this means
  • Total permutations: n! grows super-exponentially.
  • Pattern-avoiding permutations: |Avₙ(σ)| grows at most exponentially — much smaller than n!.
  • The theorem says: if you exclude even one pattern, the set shrinks dramatically from factorial to exponential growth.
  • Special cases: L(321) = L(231) = 4 (the Catalan growth rate); L(1324) ≈ 11.6 (harder to compute).

08Permutons — Limits of Permutation Sequences

Graphons as Motivation

Graphon (informal) A graphon is the limit object for sequences of dense graphs. It is a symmetric measurable function W : [0,1]² → [0,1] that encodes the "template" for making a random graph: to generate a graph on n vertices, sample n points uniformly from [0,1] and connect vertices i and j with probability W(xᵢ, xⱼ).

Graphons provide a language for saying "a sequence of graphs converges": a sequence (Gₙ) converges if for every fixed graph H, the density of H in Gₙ converges. The limit is a graphon.

Permutons

Definition — Permuton A permuton is a probability measure μ on the unit square [0,1]² with uniform marginals — both the x-projection and y-projection of μ are the uniform distribution on [0,1].

Each permutation σ of length n gives a permuton by placing mass 1/n at each point:
μ_σ = (1/n) ∑_{i=1}^{n} δ_{((i−½)/n, (σ(i)−½)/n)}
This is the permutation matrix of σ, rescaled to live in [0,1]².
Definition — Pattern Density For permutations σ of length n and pattern π of length k, the pattern density of π in σ is:
t(π, σ) = |{k-element subsets of positions realizing π in σ}| / C(n,k)
This is the probability that a random k-element subset of {1,…,n} realizes pattern π.

Why pattern density instead of Kendall tau: Kendall tau compares permutations of the same size. Pattern density gives a normalized count that works across permutations of different sizes, enabling us to compare and take limits.
Definition — Convergence of Permutation Sequences A sequence of permutations σ₁, σ₂, σ₃, … (with |σₙ| → ∞) converges to a permuton μ if for every fixed pattern π ∈ Sₖ, the pattern densities converge:
t(π, σₙ) → t(π, μ) as n → ∞
where t(π,μ) is the density of π in the limiting measure.
The big picture Permutons give a compact metric space in which sequences of permutations have convergent subsequences. This allows meaningful statements like "uniform random 321-avoiding permutations of length n converge to a specific permuton" — describing the typical shape of a large pattern-avoiding permutation.

The connection to graphons: every permutation has an associated "permutation graph," giving a bridge between the two theories. But permutons encode finer information than graphons — they remember the ordering structure, not just pairwise comparisons.

Q8The Catalan recurrence for 231-avoiding permutations: why must the elements left of n be exactly {1,…,k−1} when n is at position k?▼
Solution
Suppose some left value a > some right value b, i.e., a is at some position i < k and b is at some position j > k with a > b. We show this creates a 231 pattern in σ.
The 231 pattern: Consider positions i, k, j (left to right in σ). Values: σ(i)=a, σ(k)=n, σ(j)=b. We have b < a < n, so relative order is (a, n, b) with b < a < n. The pattern is: medium, large, small = 231. ✓
Conclusion: If σ avoids 231, no such a and b can exist. So all left values must be < all right values. Since the left has k−1 values from {1,…,n−1} and they must all be less than all n−k right values, and the sets are complementary, the left values must be exactly {1,…,k−1} and right values exactly {k,…,n−1}.

A 231 pattern forces left values to be {1,…,k−1}: (leftval, n, rightval) = (medium, large, small) = 231.

Upper bound (from Week 6): Explicit n-element realizer using rotated lexicographic orders.

Lower bound: Key lemma — in any linear extension L, at most one diagonal pair (aᵢ,bᵢ) can have bᵢ before aᵢ. Proof: if bᵢ before aᵢ AND bⱼ before aⱼ for i≠j, the chain aⱼ→bᵢ→aᵢ→bⱼ→aⱼ is a cycle in L, contradicting that L is a total order. Since all n pairs must be separated, need n extensions. dim ≥ n. ∎

HW — Due Wednesday May 13

Problem: Describe the permutations that avoid the pattern 321.

Answer: A permutation avoids 321 iff it has no decreasing subsequence of length 3, iff its longest decreasing subsequence has length ≤ 2, iff it can be partitioned into at most 2 increasing subsequences. Equivalently, it is 2-increasing (can be written as a merge of 2 increasing sequences). These are also called the "queue-sortable" permutations.

HW — Due Friday May 15

Problem: Devise a sorting machine that works iff the permutation is 321-avoiding. Use it to show the count equals the nth Catalan number.

The machine: Use 2 increasing queues (patience sort). Process each element: put it in queue 1 if it extends queue 1 (greater than its current last element), else queue 2, else FAIL. Succeeds iff 321-avoiding (shown above).

Count = Cₙ:

  1. |Avₙ(321)| = |Avₙ(231)| (both equal Cₙ by the Catalan recurrence, since all 6 length-3 patterns give the same count).
  2. |Avₙ(231)| = Cₙ by the Catalan recurrence: place n at position k, left = {1,…,k−1}, right = {k,…,n−1}, giving f(n) = Σ f(k−1)·f(n−k) = Cₙ.
  3. Bijection: Avₙ(231) ↔ Dyck paths of length 2n via stack-sort (PUSH=U, POP=D), so |Avₙ(231)| = number of Dyck paths = Cₙ.
  4. Cₙ = C(2n,n)/(n+1) via the ballot theorem / cycle lemma.

dim(Sₙ) = n
  • Sₙ: aᵢ < bⱼ iff i≠j; diagonal pairs (aᵢ,bᵢ) incomparable

Pattern Avoidance
  • σ avoids π: no k positions in σ with values in relative order of π
  • Permutation class: closed downward under pattern containment
  • Every class = Av(F) for some forbidden set F
321-Avoiding
  • ≡ no decreasing subseq of length 3
  • ≡ can be split into 2 increasing subsequences
  • Sorted by 2-queue patience sort machine
  • Count = Cₙ
Stack Sort ↔ 231-Avoiding
  • Stack sort: push if top>input, pop while top≤input
  • Succeeds iff σ avoids 231
  • Proof (⟹): failure gives (t,c,k) = 231 pattern with c>t>k
  • Stack sort bijection: PUSH=U, POP=D → Dyck path
Catalan Numbers
  • Cₙ = C(2n,n)/(n+1)
  • Recurrence: Cₙ = Σ C_{k-1}·C_{n-k}
  • = # Dyck paths of length 2n
  • = # 231-avoiding perms of [n]
  • = # 321-avoiding perms of [n]
  • P(random path ends at 0 is Dyck) = 1/(n+1)
