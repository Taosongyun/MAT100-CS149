01Why We Care

A permutation is just a rearrangement — but when you choose one uniformly at random, surprisingly deep structure emerges. This week we build the language and intuition to answer questions like:

  • If 100 prisoners each follow a cycle-tracing strategy, what is the probability they all survive?
  • If you shuffle a deck and deal cards calling out 1, 2, 3…, what is the probability you never match?
  • How many elements, on average, does a random permutation leave in place?

The answers — ln 2, 1/e, and 1 respectively — are elegant and non-obvious. Understanding why they're true is the goal of this week.

Course Context These results connect directly to the two homework problems: HW1 asks you to derive the ln 2 limit, and HW2 asks you to prove the uniformity theorem that makes the prisoners' strategy analysis valid.

02Definitions

Permutation

Definition — Permutation A permutation of the set {1, 2, …, n} is a bijection σ : {1,…,n} → {1,…,n}. In other words, σ rearranges the elements — every element appears exactly once as an input and exactly once as an output.

The set of all permutations of {1,…,n} is called the symmetric group Sₙ and has exactly n! elements.

Two ways to write a permutation

One-line notation: simply list the outputs in order: σ = σ(1) σ(2) … σ(n). For example, σ = 2 1 4 3 means σ(1)=2, σ(2)=1, σ(3)=4, σ(4)=3.

Two-line notation: write inputs on top, outputs below:

Cycles

Definition — Cycle A k-cycle is a permutation that moves k elements in a "rotation" and fixes everything else. It is written (a₁ a₂ … aₖ) and defined by:
σ(a₁) = a₂, σ(a₂) = a₃, …, σ(aₖ₋₁) = aₖ, σ(aₖ) = a₁
Every element not in {a₁,…,aₖ} is fixed: σ(x) = x.
Key Fact — Cycle Decomposition Every permutation can be written uniquely as a product of disjoint cycles (cycles that share no elements). This decomposition is unique up to the order in which we write the cycles and the starting point within each cycle. We always have (a₁ a₂ … aₖ) = (a₂ a₃ … aₖ a₁) = (a₃ … aₖ a₁ a₂) — they're the same cycle.

How to find cycles: the tracing algorithm

  1. Pick the smallest element not yet in a cycle. Call it a₁.
  2. Apply σ: a₂ = σ(a₁), a₃ = σ(a₂), and so on.
  3. Stop when you return to a₁. You've found the cycle (a₁ a₂ … aₖ).
  4. Repeat with the next smallest element not yet placed.

Fixed Points

Definition — Fixed Point An element i ∈ {1,…,n} is a fixed point of σ if σ(i) = i. In cycle notation, fixed points appear as 1-cycles: (i). Convention: we often omit 1-cycles when writing cycle notation, so (1 3)(2) and (1 3) mean the same permutation when working in S₃.
Theorem — Expected Fixed Points For a uniformly random permutation σ ∈ Sₙ, the expected number of fixed points is exactly 1, for every n ≥ 1.

Proof: Let Xᵢ = 𝟙[σ(i) = i] be the indicator that i is a fixed point. By symmetry, each element is equally likely to map to any of the n positions, so Pr(Xᵢ = 1) = 1/n. By linearity of expectation:
E[# fixed points] = E[X₁ + X₂ + … + Xₙ] = n · (1/n) = 1

Derangements

Definition — Derangement A derangement is a permutation with no fixed points — every element moves. The number of derangements of {1,…,n} is denoted Dₙ (also written !n).
Theorem — Derangement Formula By inclusion-exclusion:
Dₙ = n! · ∑ₖ₌₀ⁿ (−1)ᵏ / k! = n!(1 − 1 + 1/2! − 1/3! + … + (−1)ⁿ/n!)
As n → ∞, the probability of a derangement converges to:
Pr(derangement) = Dₙ/n! → ∑ₖ₌₀^∞ (−1)ᵏ/k! = e⁻¹ ≈ 0.3679
Why? The series ∑(−1)ᵏ/k! is the Taylor expansion of e⁻¹, and adding more terms changes the value by less than 1/n! → 0.
The Game of 13 (Treize) This is the historical origin of derangements, from de Montmort (1708). Shuffle 13 cards numbered 1–13. Flip them one at a time, calling out 1, 2, 3, …, 13 as you go. You win if any card matches the number called; you lose only if no card ever matches — that is, if the shuffle is a derangement. Since D₁₃/13! ≈ 1/e, the probability of losing is about 36.8%. The game is cleverly designed so this probability is nearly constant regardless of how many cards you use.

Long Cycles

Definition — Long Cycle A cycle in a permutation is called a long cycle if its length exceeds n/2. Note that at most one cycle can be long (two cycles each of length > n/2 would require more than n elements total).
Theorem — Long Cycle Probability (HW1) For a uniform random permutation of {1,…,n}, as n → ∞:
Pr(σ has a cycle of length > n/2) → ln 2 ≈ 0.6931
Proof idea: For a specific set S of k elements, the probability they form a single k-cycle in a random permutation of [n] is (k−1)! · (n−k)! / n! = 1/(k · C(n,k)). Summing over all sets S of size k (there are C(n,k) of them), the probability of having some k-cycle is exactly 1/k. Since at most one long cycle can exist (the events are mutually exclusive), we sum:
Pr(long cycle) = ∑_{k = ⌊n/2⌋+1}^{n} 1/k → ∫_{1/2}^{1} (1/x) dx = ln(1) − ln(1/2) = ln 2

The 100 Prisoners Problem

Setup 100 prisoners are numbered 1–100. The warden writes their names on slips and places one in each of 100 boxes (numbered 1–100). Each prisoner may open 50 boxes. They survive iff all prisoners find their own name. No communication after the process begins.

The strategy: Each prisoner starts at their own numbered box, opens it, reads the name inside, and goes to the box with that number. They repeat until they find their own name or exhaust 50 opens. This is the cycle-following strategy.

Why it works: Each prisoner follows exactly the cycle that contains their number in the warden's permutation. They succeed iff that cycle has length ≤ 50. They all succeed iff the permutation has no cycle of length > 50.
Pr(all survive) = 1 − Pr(∃ cycle of length > 50) → 1 − ln 2 ≈ 30.7%
HW 2 — Uniformity Theorem Claim: No matter how the warden assigns names to boxes, if prisoners assign box ownership uniformly at random, the resulting permutation (prisoner → name in their box) is uniformly random.

Proof: Let w : boxes → names be the warden's fixed bijection and b : prisoners → boxes be the prisoners' uniform random bijection. The composed function σ = w ∘ b maps each prisoner to the name in their box.

For any target permutation τ : prisoners → names:
Pr(σ = τ) = Pr(w ∘ b = τ) = Pr(b = w⁻¹ ∘ τ)
Since b is uniform over all n! bijections, and w⁻¹ ∘ τ is one specific bijection, this probability equals 1/n!. This holds for every τ, so σ is uniform. ∎

04Worked Examples

🟢 Easy: Decomposing σ = 2 1 3 4

Step-by-step cycle decomposition

We have σ(1)=2, σ(2)=1, σ(3)=3, σ(4)=4.

Step 1: Start at 1. Apply σ: 1 → 2 → 1. We returned to 1 after 2 steps. First cycle: (1 2).

Step 2: Next unused element is 3. Apply σ: 3 → 3. Returned immediately. Cycle: (3) — a fixed point.

Step 3: Next unused element is 4. Apply σ: 4 → 4. Fixed point: (4).

Result: σ = (1 2)(3)(4). Fixed points: {3, 4}. The permutation has 2 fixed points and one 2-cycle. It is not a derangement.

🟡 Medium: The Prisoners Problem with n = 6

Setup: σ = 3 4 5 2 1 6

Six prisoners, six boxes. The warden's permutation is σ(1)=3, σ(2)=4, σ(3)=5, σ(4)=2, σ(5)=1, σ(6)=6.

Step 1 — Find cycles:

Start at 1: 1 → 3 → 5 → 1. Cycle: (1 3 5), length 3.

Start at 2 (next unused): 2 → 4 → 2. Cycle: (2 4), length 2.

Start at 6 (next unused): 6 → 6. Cycle: (6), length 1 (fixed point).

Full decomposition: σ = (1 3 5)(2 4)(6).

Step 2 — Apply the prisoners strategy:

Each prisoner starts at their own box. With n/2 = 3 allowed opens:

  • Prisoner 1 opens boxes 1 → 3 → 5. Finds name "1" in box 5. ✓ (3 opens, exactly on the limit)
  • Prisoner 2 opens boxes 2 → 4. Finds "2" in box 4. ✓ (2 opens)
  • Prisoner 3 opens boxes 3 → 5 → 1. Finds "3" in box 1. ✓ (3 opens)
  • Prisoner 4 opens boxes 4 → 2. Finds "4" in box 2. ✓ (2 opens)
  • Prisoner 5 opens boxes 5 → 1 → 3. Finds "5" in box 3. ✓ (3 opens)
  • Prisoner 6 opens box 6. Finds "6" immediately. ✓ (1 open)

All prisoners survive! This works because the longest cycle has length 3 = n/2. If any cycle had length 4, 5, or 6, that prisoner would run out of opens.

🔴 Hard: Deriving Pr(long cycle) → ln 2

Full derivation

Setup. We want Pr(σ has a cycle of length k) for a specific k, then sum over all k > n/2.

Step 1 — Probability of a specific k-cycle. Fix a set S = {a₁,…,aₖ} of k elements. How many permutations in Sₙ have these k elements forming a single k-cycle, with the remaining n−k elements in any arrangement?

  • Number of k-cycles on S: (k−1)! (fix one element, arrange the rest)
  • Number of arrangements of remaining n−k elements: (n−k)!
  • Total favorable: (k−1)! · (n−k)!
  • Probability: (k−1)! · (n−k)! / n!

Step 2 — Probability that some k-cycle exists. There are C(n,k) choices for S. Since these events are mutually exclusive (a permutation can have at most one k-cycle on any given set S, and different sets S are disjoint events):

Pr(∃ k-cycle) = C(n,k) · (k−1)! · (n−k)! / n! = [n! / (k! · (n−k)!)] · (k−1)! · (n−k)! / n! = 1/k

Step 3 — Sum over long cycles. For k > n/2, at most one such cycle can exist, so events for different values of k are mutually exclusive:

Pr(long cycle) = ∑_{k=⌊n/2⌋+1}^{n} 1/k

Step 4 — Take the limit. As n → ∞, this Riemann sum converges to an integral:

∑_{k=⌊n/2⌋+1}^{n} 1/k ≈ ∑_{k=⌊n/2⌋+1}^{n} (1/n) · (n/k) → ∫_{1/2}^{1} (1/x) dx = [ln x]_{1/2}^{1} = 0 − ln(1/2) = ln 2

Practice Problems — Permutations, Cycles & Fixed Points
Q1Write σ = 3 1 4 2 (one-line notation on {1,2,3,4}) in cycle notation. Identify all fixed points.▼
Solution

We use the tracing algorithm. σ(1)=3, σ(2)=1, σ(3)=4, σ(4)=2.

Trace from 1: 1 → σ(1) = 3 → σ(3) = 4 → σ(4) = 2 → σ(2) = 1. We returned to 1 after visiting {1, 3, 4, 2}. This is a single 4-cycle.
All elements used: {1, 2, 3, 4} are all in the cycle. No remaining elements.

Cycle notation: (1 3 4 2)

Fixed points: none — every element moves.

Is it a derangement? Yes — no fixed points.

Q2List all derangements of {1, 2, 3}. Verify using the formula D₃ = 3!(1 − 1 + 1/2 − 1/6).▼
Solution

A derangement of {1,2,3} is a permutation where σ(1)≠1, σ(2)≠2, σ(3)≠3. Let's check all 3! = 6 permutations:

Q3Prove that the expected number of fixed points of a uniform random permutation of {1,…,5} equals 1. (Use linearity of expectation — don't just cite the theorem, carry out the proof for n = 5.)▼
Solution

Define indicator random variables: for each i ∈ {1,2,3,4,5}, let

Xᵢ = 1 if σ(i) = i, and Xᵢ = 0 otherwise.

Step 1: Find E[Xᵢ]. What is the probability that element i maps to itself? A uniform random permutation is equally likely to send i to any of the 5 positions. Exactly 1 of those 5 possibilities is σ(i) = i. So Pr(Xᵢ = 1) = 1/5, and therefore E[Xᵢ] = 1/5.
Step 2: Apply linearity of expectation. The total number of fixed points is X = X₁ + X₂ + X₃ + X₄ + X₅. Linearity of expectation says E[sum] = sum of E's, even for dependent random variables:
E[X] = E[X₁] + E[X₂] + E[X₃] + E[X₄] + E[X₅] = 5 · (1/5) = 1.

E[# fixed points] = 1 for n = 5. The same argument works for any n: n terms each equal to 1/n. ∎

Q4What is the probability that a uniform random permutation of {1, 2, 3, 4} has at least one fixed point?▼
Solution

It is easier to compute the complement: Pr(no fixed points) = Pr(derangement) = D₄/4!.

Compute D₄. Using the formula Dₙ = n! · ∑ₖ₌₀ⁿ (−1)ᵏ/k!:
D₄ = 4!(1 − 1 + 1/2 − 1/6 + 1/24) = 24 · (12/24 − 4/24 + 1/24) = 24 · 9/24 = 9.
Compute the probability.
Pr(derangement) = 9/24 = 3/8.
Pr(≥ 1 fixed point) = 1 − 3/8 = 5/8.

Pr(at least one fixed point) = 5/8 = 0.625

Q6In the 100 Prisoners problem with n = 6 (6 prisoners, each may open 3 boxes), using the cycle-following strategy: what is the exact probability that all prisoners survive?▼
Solution

All prisoners survive iff the warden's permutation has no cycle of length > 3. We need to count permutations in S₆ where every cycle has length ≤ 3.

Method: Pr(survive) = 1 − Pr(∃ cycle of length > 3) = 1 − Pr(∃ cycle of length 4, 5, or 6).
Use the formula Pr(∃ k-cycle) = 1/k, and note events for different k > n/2 = 3 are mutually exclusive:
Pr(long cycle) = 1/4 + 1/5 + 1/6 = 15/60 + 12/60 + 10/60 = 37/60.
Survival probability:
Pr(all survive) = 1 − 37/60 = 23/60.

Pr(all survive) = 23/60 ≈ 0.3833

For comparison, the large-n limit is 1 − ln 2 ≈ 0.3069. With only n=6, we're still somewhat above the limit.

Q7Verify directly that Pr(∃ cycle of length > 3 in S₆) = 1/4 + 1/5 + 1/6 by checking that 444 out of 720 permutations contain such a cycle, and confirm 444/720 = 37/60.▼
Solution
Formula gives: 1/4 + 1/5 + 1/6 = 15/60 + 12/60 + 10/60 = 37/60.
Convert to count: 37/60 × 720 = 37 × 12 = 444 permutations should contain a long cycle.
Verify the fraction: 444/720 = 222/360 = 111/180 = 37/60. ✓
Check survival count: 720 − 444 = 276 permutations have all cycles of length ≤ 3. 276/720 = 138/360 = 23/60. ✓

444/720 = 37/60 ✓

Q8How many permutations in S₆ have exactly one fixed point? What is the probability?▼
Solution

A permutation of {1,…,6} with exactly one fixed point: choose which element is fixed, then the remaining 5 elements must form a derangement.

Step 1: Choose the fixed point. There are C(6,1) = 6 choices for which element stays in place.
Step 2: Count derangements of the remaining 5 elements. D₅ = 5!(1 − 1 + 1/2 − 1/6 + 1/24 − 1/120) = 120 · (44/120) = 44.
Step 3: Multiply. Total = 6 · D₅ = 6 · 44 = 264.
Step 4: Compute probability. 264/720 = 11/30 ≈ 0.3667.

264 permutations, probability = 11/30 ≈ 0.367

Interesting: this is very close to 1/e ≈ 0.368 — and for large n, the probability of exactly one fixed point converges to exactly 1/e as well (it follows a Poisson(1) distribution).

HW 1 — Due Wednesday April 1

Let a permutation of {1,2,…,n} be chosen uniformly at random. For large n, what is the probability that it contains a cycle of length greater than n/2?

Answer: ln 2 ≈ 0.6931. The key steps are: (1) show Pr(∃ k-cycle) = 1/k for any specific k; (2) observe events for different k > n/2 are mutually exclusive; (3) sum 1/k for k from ⌊n/2⌋+1 to n; (4) recognize this Riemann sum converges to ∫_{1/2}^{1}(1/x)dx = ln 2. HW 2 — Due Friday April 3

Write up a theorem and proof: no matter how the warden assigns names to boxes, if prisoners assign box ownership uniformly at random, the resulting permutation is uniformly random.

Answer: The composed map σ = w ∘ b (warden's bijection composed with prisoners' uniform random bijection) is uniform because composing with a fixed bijection is a bijection on the set of bijections — it maps each of the n! choices for b to a distinct choice for σ, preserving the uniform distribution.
