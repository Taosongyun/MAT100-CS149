01Why We Care

So far we have been sorting — taking a completely unknown ordering and identifying it by making comparisons. But in many real situations, we already know some of the order. We know A beats B, B beats C, but we don't know how D compares to any of them.

This partial information is exactly what a partial order captures. Partial orders let us:

  • Represent what a sorting algorithm knows so far at any point in its execution.
  • Understand when a sorting algorithm is "done" — when the poset has only one linear extension.
  • Analyze the state of algorithms like the shove algorithm as a structured mathematical object.

The Hasse diagram is the visual language for partial orders — the same way a graph is the visual language for pairwise relationships.

02Definitions

Partial Orders and Posets

Definition — Partial Order A partial order on a set P is a relation ≤ satisfying three axioms for all x, y, z ∈ P:
  1. Reflexive: x ≤ x — every element is related to itself.
  2. Antisymmetric: if x ≤ y and y ≤ x, then x = y — no two distinct elements can each be ≤ the other.
  3. Transitive: if x ≤ y and y ≤ z, then x ≤ z — the relation "chains through".
A set equipped with a partial order is called a partially ordered set or poset.

We write x < y (strict) when x ≤ y and x ≠ y. We say x and y are incomparable (written x ∥ y) if neither x ≤ y nor y ≤ x.
Definition — Total Order (Linear Order) A partial order is a total order (or linear order) if every pair of elements is comparable: for all x, y, either x ≤ y or y ≤ x. There are no incomparable pairs.

Example: The usual ≤ on integers is a total order. The divisibility relation on {1,2,3,4,6,12} is a partial order but not a total order (e.g. 4 and 6 are incomparable — neither divides the other).
Partial Order vs Directed Graph A directed graph (digraph) also has a relation between elements (edges). The key differences:
  • A digraph is not required to be reflexive, antisymmetric, or transitive.
  • A partial order must satisfy all three. Transitivity in particular is powerful — it means if you can "follow the order" from x to y to z, then x ≤ z automatically.
  • The Hasse diagram of a poset looks like a directed graph, but we draw it without arrows (direction is given by vertical position) and without redundant edges (those implied by transitivity).

Standard Examples of Posets

Poset Zoo
  • Divisibility: Dₙ = divisors of n, ordered by "divides". a ≤ b iff a | b. Example: D₁₂ = {1,2,3,4,6,12}.
  • Subset inclusion: Bₙ = all subsets of {1,…,n}, ordered by ⊆. Called the Boolean lattice.
  • Integers: (ℤ, ≤) — total order.
  • Sorting states: permutations ordered by "which states can reach which" under an algorithm — our main example this week.
  • String prefix: strings ordered by "is a prefix of". "cat" ≤ "catch".
  • Antichain: a set with NO comparable pairs — every element is incomparable to every other. The relation is just equality.

How to Check if a Relation is a Partial Order

The Three-Step Check Given a relation R on a set P, check each axiom:
  1. Reflexive: Is x R x for every x? If any element is not related to itself → FAIL.
  2. Antisymmetric: If you find x R y and y R x with x ≠ y → FAIL.
  3. Transitive: If x R y and y R z, must x R z? If any triple violates this → FAIL.
If all three pass: it's a partial order. ✓

Hasse Diagrams

Definition — Cover Relation In a poset, we say y is covered by x (written y ⋖ x) if y < x and there is no element z with y < z < x. In other words, x is the "next step up" from y — nothing in between.
Definition — Hasse Diagram The Hasse diagram of a poset (P, ≤) is drawn as follows:
  • Each element of P is a node (dot).
  • Draw an upward edge from y to x whenever y ⋖ x (y is covered by x).
  • No arrows — direction is encoded by height: higher = larger in the order.
  • No redundant edges — edges implied by transitivity are omitted. If a < b < c, we draw a-b and b-c but NOT a-c (it's implied).
  • No self-loops — reflexivity is understood, not drawn.
How to read a Hasse diagram
  • x ≤ y iff you can get from x to y by following edges upward.
  • Minimal elements are at the bottom, maximal elements at the top.
  • Two elements at the same height with no path between them are incomparable.

Linear Extensions

Definition — Linear Extension A linear extension of a poset (P, ≤) is a total order ≤* on the same set that is consistent with ≤: if x ≤ y in P, then x ≤* y.

In plain English: a linear extension is a way to put all elements in a single line so that you never violate any of the poset's relations.

We write e(P) for the number of linear extensions of P.
Example — Linear extensions of D₄ = {1, 2, 4} Relations: 1 | 2 and 1 | 4 and 2 | 4. So 1 < 2 < 4 — this is already a chain! Only one linear extension: 1, 2, 4.

Now consider the poset {a, b, c} with only the relation a < b (c is incomparable to both). Linear extensions: any ordering consistent with a before b: a, b, c — a, c, b — c, a, b. So e(P) = 3.

The Link to Sorting

Key Insight — Sorting as Refining a Poset When a sorting algorithm makes comparisons, it is building a poset:
  • Start: the poset is an antichain — all elements are incomparable (we know nothing about the order).
  • Each comparison: learning that aᵢ < aⱼ adds the relation aᵢ < aⱼ to the poset (plus any relations implied by transitivity).
  • End: the algorithm is done when the poset has exactly one linear extension — the full sorted order is now determined.
This is why the lower bound is ⌈log₂(n!)⌉: we start with n! linear extensions (all possible orderings) and must reduce to 1 using binary comparisons.
When all linear extensions agree At any point during sorting, the poset encodes everything the algorithm knows. Any linear extension of the current poset is a valid sorted output — you can't tell which one is "right" yet. When there's only one linear extension left, all algorithms that consistently respect the poset will give the same answer. That's the moment sorting is complete.

04Worked Examples

Is This a Partial Order? — Several Examples

Example 1 — Divisibility on D₁₂ = {1, 2, 3, 4, 6, 12}

Relation: a ≤ b iff a divides b.

AxiomCheckResult
ReflexiveDoes every number divide itself? Yes: a | a always.✓ Pass
AntisymmetricIf a | b and b | a with a ≠ b? For positive integers, a | b and b | a implies a = b.✓ Pass
TransitiveIf a | b and b | c, then a | c? Yes: if b = ka and c = lb, then c = kla.✓ Pass

Conclusion: D₁₂ is a poset. Is it a total order? No — for example, 4 ∥ 6 (neither divides the other).

Example 2 — "Strictly less than" on integers: a R b iff a < b
AxiomCheckResult
ReflexiveIs a < a? No — 5 < 5 is false.✗ Fail
Antisymmetric(Would pass — strict < can't go both ways)—
Transitive(Would pass)—

Conclusion: NOT a partial order (fails reflexivity). Note: strict < is a strict partial order — a different but related concept. The standard definition of partial order uses ≤ (non-strict).

Example 3 — "Has the same length" on strings

Relation: s R t iff |s| = |t| (same length).

AxiomCheckResult
Reflexive|s| = |s|. Always true.✓ Pass
AntisymmetricIf |s| = |t| and |t| = |s|, must s = t? No! "cat" and "dog" have the same length but are different strings.✗ Fail
Transitive(Would pass)—

Conclusion: NOT a partial order (fails antisymmetry). This relation is an equivalence relation, not a partial order.

Example 4 — "Is a prefix of" on strings: s R t iff s is a prefix of t
AxiomCheckResult
ReflexiveIs every string a prefix of itself? Yes.✓ Pass
AntisymmetricIf s is a prefix of t AND t is a prefix of s? Then |s| ≤ |t| and |t| ≤ |s|, so |s| = |t|, and since they share the same prefix of that length, s = t.✓ Pass
TransitiveIf s is a prefix of t and t is a prefix of u, is s a prefix of u? Yes.✓ Pass

Conclusion: Prefix relation IS a partial order. Note: "cat" ∥ "cap" (neither is a prefix of the other).

The State Poset for the Shove Algorithm (n=4)

The Shove Algorithm Given a permutation of {1,2,3,4}: find any item k that is NOT in position k. Move item k to position k, shifting the items between its old and new position by one step. Repeat until sorted.

Example: In [3, 1, 4, 2]: item 3 is at position 1, belongs at position 3. Move 3 to position 3, shift items at positions 2 and 3 left: [1, 4, 3, 2]. (Items 1 and 4 shift from positions 2,3 to positions 1,2.)
Theorem — The Shove Algorithm Always Terminates No matter what choices the algorithm makes, it always reaches the sorted permutation (1,2,3,4) in finitely many steps.
Proof — State Graph is a DAG

Define the state graph: nodes are the 24 permutations of {1,2,3,4}, and there is a directed edge from permutation σ to permutation τ if a single shove takes σ to τ.

We verified computationally that this directed graph has no cycles — it is a DAG (directed acyclic graph). Since every path in the graph strictly moves forward (no cycles), any sequence of shoves must eventually reach a state with no valid shoves: the sorted permutation (1,2,3,4). ∎

Monovariant: Define f(σ) = the length of the longest path from σ to (1,2,3,4) in the state graph. Every shove strictly decreases f(σ) (verified: 0 violations across all 66 edges). Since f is a non-negative integer that strictly decreases, the algorithm terminates. ∎

The State Poset — 8 Levels, 24 Permutations

The state poset orders permutations by reachability: σ ≤ τ iff you can reach τ from σ by a sequence of shoves. The sorted permutation (1,2,3,4) is the unique minimum element.

The level of each permutation = the longest path to (1,2,3,4) in the state graph:

Practice Problems — Partial Orders & Hasse Diagrams
Q1Is the relation "x R y iff x² ≤ y²" a partial order on the integers ℤ? Check all three axioms.▼
Solution
Reflexive: Is x² ≤ x²? Yes, always. ✓
Antisymmetric: If x² ≤ y² and y² ≤ x², then x² = y², so |x| = |y|. But this means x = y OR x = -y. For example, 3 R (-3) and (-3) R 3 (since 9 ≤ 9 in both directions) but 3 ≠ -3. ✗ FAIL

NOT a partial order — antisymmetry fails. Counterexample: 3 and -3.

Q2Draw the Hasse diagram of D₃₀ = {1, 2, 3, 5, 6, 10, 15, 30} ordered by divisibility. List all cover relations.▼
Solution
Find cover relations: a ⋖ b iff a | b and no c with a | c | b (strictly between).
  • 1⋖2, 1⋖3, 1⋖5 (nothing between 1 and any prime)
  • 2⋖6 (2|6, between: need c with 2|c|6 and 2<c<6. Only 3 divides 6 and is between, but 2∤3. ✓), 2⋖10, 3⋖6, 3⋖15, 5⋖10, 5⋖15
  • 6⋖30 (6|30, between: need 6|c|30 with 6<c<30. Divisors of 30 between 6 and 30: {10,15}. Does 6|10? No. Does 6|15? No. ✓), 10⋖30, 15⋖30

Cover relations: 1⋖2, 1⋖3, 1⋖5, 2⋖6, 2⋖10, 3⋖6, 3⋖15, 5⋖10, 5⋖15, 6⋖30, 10⋖30, 15⋖30. (12 cover relations)

The diagram has 4 levels: {1} at bottom, primes {2,3,5} next, then {6,10,15}, then {30} at top.

12 cover relations. Structure: diamond-like with 4 levels.

Q3How many linear extensions does the poset on {a, b, c, d} with only the relations a < c and b < d have?▼
Solution

We have two independent chains: a < c and b < d. The elements {a,b,c,d} must be arranged so that a comes before c AND b comes before d. These are the only constraints.

Count: Total orderings of 4 elements = 4! = 24. We need the fraction consistent with both constraints.
Method: Choose positions for {a,c}: there are C(4,2) = 6 ways to pick 2 positions out of 4. In each case, a must occupy the smaller position and c the larger — so only 1 valid arrangement out of 2. Similarly for {b,d}: the remaining 2 positions are forced (b before d). So total = C(4,2) = 6.

e(P) = 6 linear extensions

Equivalently: 4! / (2 × 2) = 24/4 = 6 — divide by 2 for each chain (half of unrestricted orderings satisfy a<c, and independently half satisfy b<d).

Q4In the poset with elements {1,2,3,4} and relations 1<3 and 2<3 and 3<4 (so 1 and 2 are incomparable), find all linear extensions.▼
Solution

Constraints: 1 before 3, 2 before 3, 3 before 4. Element 4 must be last. Elements 1 and 2 can be in either order, but both must precede 3.

Fix positions of 4: 4 must be in position 4.
Fix positions of 3: 3 must be in position 3 (since both 1 and 2 must come before 3, and 4 must come after). Actually 3 could be in position 3, since we need exactly 2 elements before 3 (which are 1 and 2). Wait — 3 must have both 1 and 2 before it, so 3 can only be in position 3 or later. Since 4 is in position 4, 3 must be in position 3.
Order 1 and 2: They can appear in either order in positions 1 and 2.

Linear extensions: (1,2,3,4) and (2,1,3,4).

e(P) = 2 linear extensions: (1,2,3,4) and (2,1,3,4)

Q5Apply one step of the shove algorithm to [2,3,1,4]. Identify which items can be shoved and what each shove produces.▼
Solution

Current permutation: [2,3,1,4]. Item k is misplaced if it's not at position k.

  • Item 1 is at position 3 (should be at position 1) → misplaced ✓
  • Item 2 is at position 1 (should be at position 2) → misplaced ✓
  • Item 3 is at position 2 (should be at position 3) → misplaced ✓
  • Item 4 is at position 4 (should be at position 4) → correctly placed
Shove item 1 (from pos 3 to pos 1): move item 1 to position 1, shift items at positions 1 and 2 right by one. Before: [2,3,1,4]. After: [1,2,3,4]. Result: [1,2,3,4] — sorted! ✓
Shove item 2 (from pos 1 to pos 2): move item 2 to position 2, shift item at position 2 left. Before: [2,3,1,4]. After: [3,2,1,4]. Result: [3,2,1,4].
Shove item 3 (from pos 2 to pos 3): move item 3 to position 3, shift item at position 3 left. Before: [2,3,1,4]. After: [2,1,3,4]. Result: [2,1,3,4].

Three possible shoves: item 1 → [1,2,3,4]; item 2 → [3,2,1,4]; item 3 → [2,1,3,4]

Note: shoving item 1 immediately sorts the array — the best possible choice. The algorithm doesn't have to make the best choice; any valid shove is allowed.

Q6In the state poset for the shove algorithm (n=4), what level is the permutation [4,3,2,1]? How many shoves does it take at minimum to sort it?▼
Solution

Looking up [4,3,2,1] = (4,3,2,1) in the state poset rank table: it appears at Level 5.

The level = longest path to sorted = maximum number of shoves needed in the worst case. The minimum number of shoves is the length of the shortest path to (1,2,3,4).

Find a short path from (4,3,2,1) to (1,2,3,4):
  • Shove item 1 (at pos 4, belongs at pos 1): [1,4,3,2] — level 3.
  • Shove item 2 (at pos 4, belongs at pos 2): [1,2,4,3] — level 1.
  • Shove item 3 (at pos 4, belongs at pos 3): [1,2,3,4] — level 0. ✓

3 shoves suffice! But the algorithm might make worse choices and take up to 5 shoves (since level = 5).

Level 5. Minimum shoves to sort = 3; maximum (worst choices) = 5.

Q7True or false, and explain: "Every total order is a partial order, but not every partial order is a total order."▼
Solution

True.

Every total order is a partial order: A total order satisfies reflexivity, antisymmetry, and transitivity (the three axioms of a partial order), AND the extra condition that all pairs are comparable. So it satisfies all the partial order axioms. ✓
Not every partial order is a total order: We need a counterexample — a partial order with at least one incomparable pair. Example: D₁₂ with divisibility. Elements 4 and 6 satisfy neither 4|6 nor 6|4 — they are incomparable. So D₁₂ is a partial order that is NOT a total order. ✓

True. A total order is a partial order with the extra requirement that all pairs are comparable.

Q8A sorting algorithm has discovered the relations a<b, b<c, and d<c (and nothing else about the 4 elements {a,b,c,d}). How many linear extensions remain? What does this tell you about how many more comparisons are needed?▼
Solution
Identify the poset: Relations are a<b<c and d<c. Elements a,b are incomparable to d. The element d is only known to be below c. We need to count orderings of {a,b,c,d} consistent with: a before b, b before c, d before c.
Count linear extensions: c must be last (both b and d must precede c, and a must precede b). So c is in position 4. The remaining positions {1,2,3} must have a before b, and d can go anywhere in {1,2,3}.
Enumerate: Orderings of {a,b,d} with a before b: (a,b,d), (a,d,b), (d,a,b) — that's 3 orderings. Each gives a full linear extension: (a,b,d,c), (a,d,b,c), (d,a,b,c).

3 linear extensions remain.

We need to reduce from 3 to 1. Since ⌈log₂(3)⌉ = 2, at least 2 more comparisons are needed. One comparison (e.g. a vs d) can split 3 into 1 and 2, then one more resolves the remaining 2. So exactly 2 more comparisons suffice.

06Homework Problems

HW — Due Wednesday April 15

Problem: n items in a row. Choose one uniformly at random, remove it and all items to its left or right (whichever is the smaller set). Repeat until all gone. How many procedures does this require on average?

Setup: If item at position k (1-indexed) is chosen: remove k items (item + left side) if left side ≤ right side (k ≤ (n+1)/2), leaving n−k items; otherwise remove n−k+1 items, leaving k−1 items.

Recurrence: Let E(n) = expected number of steps for n items.

E(0) = 0 E(n) = (1/n) · ∑_{k=1}^{n} [1 + E(remaining after choosing k)]

Exact values: E(1)=1, E(2)=2, E(3)=8/3≈2.67, E(4)=10/3≈3.33, E(5)=19/5=3.8.

Answer: E(n) = Θ(log n). The expected number of procedures grows logarithmically. Intuitively: on average, each step removes about n/4 to n/2 items (since we remove the smaller half), so the list shrinks exponentially fast — roughly halving every step or two.

HW — Due Friday April 17

Problem: Does the shove algorithm always terminate, even with bad choices?

Answer: Yes, it always terminates.

Proof: The state graph on the 4! = 24 permutations of {1,2,3,4} (with edges given by valid shoves) is a directed acyclic graph (DAG) — verified: it has 0 directed cycles. Since there are no cycles, any sequence of moves must eventually reach a state with no available shoves — namely, the sorted permutation (1,2,3,4). The algorithm terminates in at most 7 shoves (the maximum level in the state poset). ∎

For general n: define f(σ) = the length of the longest path from σ to the sorted permutation in the state graph. One can show f strictly decreases with every shove — making it a valid monovariant — which proves termination for any n.

Week 3 Cheat Sheet — Partial Orders & Hasse Diagrams
Partial Order — 3 Axioms
  • Reflexive: x ≤ x for all x
  • Antisymmetric: x≤y and y≤x ⟹ x=y
  • Transitive: x≤y and y≤z ⟹ x≤z
Key Vocabulary
  • x ∥ y: incomparable (neither ≤ the other)
  • y ⋖ x: x covers y (x is just above y)
  • total order: all pairs comparable
  • antichain: no two elements comparable
  • linear extension: total order consistent with the poset
Hasse Diagram Rules
  • Only draw cover relations (not transitive consequences)
  • Higher position = larger in the order
  • No arrows, no self-loops
  • x ≤ y iff you can walk upward from x to y
Standard Poset Examples
  • Dₙ: divisors of n by divisibility
  • Bₙ: subsets of {1..n} by ⊆
  • ℤ, ≤: integers — total order
  • Prefix order: strings by prefix relation
Sorting Connection
  • Start: antichain (n! extensions)
  • Each comparison: adds one relation
  • Done: exactly 1 linear extension remains
  • Lower bound: ⌈log₂(n!)⌉ comparisons
State Poset (n=4 shove)
  • 24 permutations, 8 levels, 34 Hasse edges
  • Minimum: 1234. Max level: 2341, 3412, 3421, 4123, 4312
  • Algorithm terminates: state graph is a DAG
  • E(n) for item removal = Θ(log n)
