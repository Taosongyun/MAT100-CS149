02Definitions

Graded Posets and Rank Functions

Definition — Graded Poset (or Ranked Poset) A finite poset P is graded (or ranked) if there exists a function rk : P → ℤ≥0 — called a rank function — satisfying all three conditions:
  1. rk(x) = 0 whenever x is minimal in P.
  2. y ⋖ x ⟹ rk(x) = rk(y) + 1 — every cover step increases rank by exactly 1.
  3. All maximal elements share the same rank h(P) − 1.
Why condition (iii) is essential Without condition (iii), the disjoint union of a 2-chain {a<b} and a 3-chain {c<d<e} would satisfy (i) and (ii): rk(a)=rk(c)=0, rk(b)=rk(d)=1, rk(e)=2. But b is maximal at rank 1 and e is maximal at rank 2 — different ranks. This poset has maximal chains of different lengths (2 and 3), which intuitively means it has no clean level structure.

Condition (iii) forces all maximal chains to have length exactly h(P), making the equivalence with the level-partition definition clean.
Equivalent Condition — All maximal chains have equal length A poset P is graded if and only if all maximal chains have the same length.

Clean consequence: In a graded poset, every maximal chain has length exactly h(P).

Standard graded examples

Standard Graded Posets
  • Bₙ (subsets of {1,…,n} by ⊆): rk(S) = |S|, with h = n+1 levels.
  • Dₙ (divisors of n by divisibility): rk(d) = Ω(d) = number of prime factors of d counted with multiplicity. E.g. in D₁₂: rk(1)=0, rk(2)=rk(3)=1, rk(4)=rk(6)=2, rk(12)=3.
  • Any chain [n]: rk(i) = i − 1.

How to test "Is this poset graded?" — two reliable tests

Test A — Maximal chains Compute the length of every maximal chain. If they are all equal, the poset is graded; if not, it is not.

Shortcut: Find the shortest and longest maximal chain. If they differ → not graded. If equal, you can stop.
Test B — Try a rank function (detect conflicts) Pick any minimal element, set rk = 0. Walk up the cover relations, assigning rk(x) = rk(y) + 1 for each cover y ⋖ x. If you ever assign two conflicting values to the same element, the poset is not graded. Otherwise it is, and you have built the rank function.

A conflict means: element z receives rank k from one path and rank k' ≠ k from another path.

What is special about graded posets?

Special Properties of Graded Posets
  • Clean level structure: Elements partition into "levels" P_0, P_1, …, P_h where P_k = {x : rk(x) = k}. All cover edges go between consecutive levels.
  • Each level is an antichain: If x, y ∈ P_k with x < y, then rk(y) > rk(x) — but both have rank k, contradiction. So no two elements in the same level are comparable.
  • Rank generating function: The polynomial F(q) = ∑_k |P_k| · qᵏ often has beautiful algebraic properties.
  • Height = max rank: h(P) = max rk(x) + 1 = number of levels.

Height, Width, and Antichains

Definitions — Height, Width, Chain, Antichain
  • A chain in P is a subset of pairwise comparable elements.
  • An antichain in P is a subset of pairwise incomparable elements.
  • The height h(P) = size of the longest chain in P.
  • The width w(P) = size of the largest antichain in P.
Note on height convention: height counts elements, not edges. A chain with 4 elements has height 4, not 3.

Convex Subsets

Definition — Convex Subset (Order-Convex / Interval) A subset C ⊆ P is convex (also called order-convex or an interval) if: whenever x, y ∈ C and z ∈ P satisfies x ≤ z ≤ y, then z ∈ C.

Plain English: C is convex if it contains everything "between" any two of its elements.

This matches the real-line case: convex subsets of (ℤ, ≤) are intervals like {3,4,5,6}, never {3,5} (the 4 in between is missing).
Convex and Non-Convex Examples in D₁₂
  • {2, 4, 6, 12} in D₁₂: Is it convex? Check all pairs. Between 2 and 12: elements z with 2|z|12 are {4, 6} — both in the set. ✓ Between 2 and 4: nothing between. ✓ Between 2 and 6: nothing between (3 is not above 2 in D₁₂ since 2∤3). ✓ Convex.
  • {2, 12} in D₁₂: Between 2 and 12 lies 4 (since 2|4|12), but 4 ∉ {2,12}. Not convex.
  • {b, c} in the V-poset (a<b, a<c): b and c are incomparable, so there is no z with b≤z≤c or c≤z≤b. The condition is vacuously satisfied. Convex.
  • In any chain, convex subsets are exactly the contiguous intervals.

Kendall Tau Distance

Definition — Kendall Tau Distance For two permutations σ, π of {1,2,…,n}, the Kendall tau distance τ(σ,π) is the number of pairs {i,j} ⊆ {1,…,n} on which σ and π disagree about which element comes first.
Three Equivalent Viewpoints
  1. Inversions: τ(σ, id) = #{(i,j) : i < j,\ σ(i) > σ(j)} = number of inversions of σ.
  2. Adjacent transpositions: τ(σ, π) equals the minimum number of adjacent swaps needed to turn σ into π. So bubble sort uses exactly τ(σ, id) swaps.
  3. Permutahedron distance: Vertices of the permutahedron Πₙ are permutations; an edge connects σ and π iff τ(σ,π) = 1. So τ is the graph distance on the 1-skeleton of Πₙ.
Example — Computing τ(3142, id)

σ = [3,1,4,2]. Count pairs (i,j) with i<j and σ(i) > σ(j):

  • (1,2): σ(1)=3 > σ(2)=1 ✓ — inversion
  • (1,3): σ(1)=3 < σ(3)=4 ✗
  • (1,4): σ(1)=3 > σ(4)=2 ✓ — inversion
  • (2,3): σ(2)=1 < σ(3)=4 ✗
  • (2,4): σ(2)=1 < σ(4)=2 ✗
  • (3,4): σ(3)=4 > σ(4)=2 ✓ — inversion

τ(3142, id) = 3. Bubble sort would use exactly 3 swaps to sort [3,1,4,2].

Mirsky's Theorem: Decomposing P into Antichains

Mirsky's Theorem Every finite poset P can be partitioned into exactly h(P) antichains A₁, A₂, …, A_{h(P)} by setting:
Aₖ = {x ∈ P : length of the longest chain ending at x equals k}
Moreover, this partition has the property that each |Aₖ| ≤ w(P), and therefore:
|P| = |A₁| + |A₂| + … + |A_{h(P)}| ≤ h(P) · w(P)
Proof — Three steps

Step 1: The Aₖ partition P. Every element x has a well-defined "longest chain ending at x" of some length k. So x belongs to exactly one Aₖ, and together they cover all of P.

Step 2: Each Aₖ is an antichain. Suppose x, y ∈ Aₖ with x < y. Let C be a longest chain ending at x, with length k. Extending C by appending y gives a chain of length k+1 ending at y. But y ∈ Aₖ means the longest chain ending at y has length k — contradiction. Therefore no two elements of Aₖ are comparable. ∎

Step 3: Bound |P|. There are h(P) antichains, each of size at most w(P). So:

|P| = ∑_{k=1}^{h(P)} |Aₖ| ≤ h(P) · w(P) ∎

Erdős–Szekeres Lemma: Full Walkthrough

Lemma (Erdős–Szekeres, 1935) Every sequence of rs + 1 distinct real numbers contains either an increasing subsequence of length at least r+1 or a decreasing subsequence of length at least s+1.

Equivalently: any sequence of length strictly greater than rs contains an increasing subsequence of length > r or a decreasing subsequence of length > s.

Special case r = s = n: any sequence of n² + 1 distinct reals has a monotone subsequence of length at least n + 1.
Connection to |P| ≤ h·w Define a poset P on the sequence: aᵢ ≤_P aⱼ iff i ≤ j AND aᵢ ≤ aⱼ. Then:
  • Chains in P = increasing subsequences. So h(P) = LIS length.
  • Antichains in P = decreasing subsequences. So w(P) = LDS length.
  • The Erdős–Szekeres lemma is precisely |P| ≤ h · w applied here: if |P| = rs+1 ≤ h·w, then h > r or w > s.
Pigeonhole Proof via (u,d) pairs

For each element aᵢ, define:

  • uᵢ = length of the longest increasing subsequence ending at aᵢ
  • dᵢ = length of the longest decreasing subsequence ending at aᵢ

Claim: all pairs (uᵢ, dᵢ) are distinct.

If i < j and aᵢ < aⱼ: any increasing subseq ending at aᵢ extends to one ending at aⱼ, so uⱼ > uᵢ — pairs differ in first coordinate. If i < j and aᵢ > aⱼ: similarly dⱼ > dᵢ. Either way (uᵢ, dᵢ) ≠ (uⱼ, dⱼ).

Pigeonhole: We have rs+1 distinct pairs. If all uᵢ ≤ r and all dᵢ ≤ s, only rs pairs from {1,…,r}×{1,…,s} are available — contradiction. So some uᵢ ≥ r+1 or some dᵢ ≥ s+1. ∎

Counting Labellings

Definition — Labelled Poset A labelled poset is a poset whose elements are the numbers {1, 2, …, n}. The Hasse diagram has a specific shape (the unlabelled structure), but the labels are assigned to the nodes.

Two labellings of the same shape are considered the same if an automorphism of the diagram maps one to the other.
Theorem — Labelling Count The number of distinct labelled posets having a given shape P on n elements is:
number of labellings = n! / |Aut(P)|
where |Aut(P)| is the size of the automorphism group of P — the number of order-preserving bijections from P to itself.

Why? The symmetric group Sₙ acts on labellings by permuting labels. Two labellings are equivalent iff some automorphism of P relates them. The number of distinct labellings = number of orbits = |Sₙ| / |Aut(P)| = n!/|Aut(P)|.

How to find |Aut(P)| An automorphism of P is an order-preserving bijection P→P. To find them:
  • Elements with the same "structural role" in the Hasse diagram can be swapped.
  • The top and bottom elements (if unique) are always fixed.
  • Elements at the same level that have identical upward and downward neighborhoods can be swapped.
  • For many posets, |Aut| = k! where k elements are "structurally identical."

The Paint Dispenser Problem — Full Proof

HW April 23 — The Problem A paint machine dispenses a Uniform[0,1] amount of paint per push. An electric eye stops it when the can (capacity 1 gallon) is full. What is the expected number of button pushes?
Theorem The expected number of pushes is exactly e ≈ 2.71828.
Full Proof — Three key steps

Setup. Let U₁, U₂, … be i.i.d. Uniform[0,1] random variables (the amounts dispensed). Define N = min{k : U₁ + U₂ + ··· + Uₖ ≥ 1} — the number of pushes needed.

Step 1: Compute Pr(N > k).

Pr(N > k) = Pr(U₁ + ··· + Uₖ < 1) — we need more than k pushes iff the first k dispensings didn't fill the can.

This probability equals the volume of the standard k-simplex:

Pr(N > k) = Vol{(x₁,…,xₖ) : xᵢ ≥ 0, x₁+···+xₖ < 1} = 1/k!

Proof of the volume formula by induction:

  • Base k=1: Vol{x : 0 ≤ x < 1} = 1 = 1/1!. ✓
  • Inductive step: Fix xₖ = t ∈ [0,1]. The remaining (k−1)-simplex has volume (1−t)^{k−1}/(k−1)!. Integrate over t:
Vol_k = ∫₀¹ (1−t)^{k−1} / (k−1)! dt = [−(1−t)^k / (k·(k−1)!)]₀¹ = 0 − (−1/k!) = 1/k! ✓

Step 2: Use the tail sum formula.

For any non-negative integer-valued random variable N:

E[N] = ∑_{k=0}^∞ Pr(N > k)

Why? N = ∑_{k=0}^∞ 𝟙[N > k], so E[N] = ∑_{k=0}^∞ Pr(N > k) by linearity.

Step 3: Substitute and recognize the Taylor series.

E[N] = ∑_{k=0}^∞ 1/k! = 1/0! + 1/1! + 1/2! + 1/3! + ··· = 1 + 1 + 1/2 + 1/6 + ··· = e ∎

The last equality is the Taylor series of eˣ evaluated at x=1.

Why does this make intuitive sense? The expected value is just above 2 — you almost always need at least 2 pushes (since 1 uniform random variable is less than 1 with probability 1, meaning 1 push is never enough). With 2 pushes you fill the can about 50% of the time. The extra probability of needing 3, 4, … pushes adds up exactly to e−2 ≈ 0.718.

The connection to Pr(N=1)=0 is immediate: a single Uniform[0,1] variable is strictly less than 1 with probability 1, so you always need at least 2 pushes.
De Finetti's Theorem — Mentioned in lecture An infinite sequence of random variables is exchangeable — meaning the joint distribution is unchanged by any finite permutation of the indices — if and only if the variables are conditionally independent given some latent random variable. In other words, exchangeable sequences are always i.i.d. mixtures.

The connection to this week: when sampling random linear extensions of a poset uniformly, the resulting sequence of elements has exchangeability properties that depend on the poset structure. De Finetti's theorem gives a structural decomposition of such distributions.

Q4How many labellings does the "fence" poset on 4 elements have? The fence has cover relations: a⋖b, c⋖b, c⋖d (so b has two elements below it, d is a second top element).▼
Solution
Find |Aut(P)|. An automorphism must preserve all cover relations. The cover relations are a⋖b, c⋖b, c⋖d. Element b is covered by both a and c; d is covered only by c. Is there any non-identity automorphism? If we try to swap a and d: a covers b (via a⋖b) but d covers d... wait, d is above c, not below. Let's be careful: a is a minimal element below b only; d is a maximal element above c only; c covers both b and d. No symmetry maps a to d while preserving the structure (a is below b, d is above c — completely different roles). |Aut(P)| = 1.
Labellings = 4!/1 = 24.

24 labellings — the fence has a trivial automorphism group.

Q5Prove that |P| ≤ h(P)·w(P) for every finite poset P. (This is HW April 22.)▼
Solution

We prove this using the Mirsky decomposition.

Define: For each x ∈ P, let rk(x) = length of the longest chain ending at x. Note rk(x) ∈ {1,…,h(P)}.
Partition P: Let Aₖ = {x ∈ P : rk(x) = k} for k = 1,…,h(P). These sets partition P.
Each Aₖ is an antichain: Suppose x, y ∈ Aₖ with x < y. The longest chain ending at x has length k; appending y gives a chain of length k+1 ending at y. But rk(y) = k — contradiction. So no two elements of Aₖ are comparable. ✓
Bound the size: Since each Aₖ is an antichain, |Aₖ| ≤ w(P). Summing over all h(P) levels: |P| = ∑|Aₖ| ≤ h(P)·w(P). ∎

|P| ≤ h(P)·w(P). Proof via Mirsky decomposition — partition P into h(P) antichains of size ≤ w(P).

Q6Compute τ(σ, id) for σ = [4,1,3,2] using all three viewpoints: inversions, bubble sort swaps, and permutahedron distance.▼
Solution
Viewpoint 1 — Inversions: Count pairs (i,j) with i<j and σ(i)>σ(j):
(1,2): 4>1 ✓; (1,3): 4>3 ✓; (1,4): 4>2 ✓; (2,3): 1<3 ✗; (2,4): 1<2 ✗; (3,4): 3>2 ✓.
Total: 4 inversions.
Viewpoint 2 — Bubble sort: Each pass swaps adjacent elements that are out of order. Trace [4,1,3,2]:
Pass 1: 4↔1→[1,4,3,2]; 4↔3→[1,3,4,2]; 4↔2→[1,3,2,4]. (3 swaps)
Pass 2: 3↔2→[1,2,3,4]. (1 swap)
Total: 4 swaps. ✓
Viewpoint 3 — Permutahedron: τ = graph distance = minimum steps on the permutahedron's 1-skeleton from [4,1,3,2] to [1,2,3,4]. This equals the number of inversions = 4 steps. ✓

τ([4,1,3,2], id) = 4. Confirmed by all three viewpoints.

Q7In D₁₂, is the subset {1, 2, 6} convex? Is {4, 6, 12} convex? Justify both answers.▼
Solution
{1, 2, 6}: Check all pairs x,y in the set — is everything between them also in the set?
Pair (1,6): elements z with 1|z|6 in D₁₂: z ∈ {2, 3, 6}. But 3 ∉ {1,2,6}! So 3 is between 1 and 6 but missing. NOT convex.
{4, 6, 12}: Check all pairs:
Pair (4,12): z with 4|z|12: z ∈ {12}... actually need 4|z and z|12 and 4<z<12. That's z ∈ {}, since no divisor of 12 strictly between 4 and 12 is divisible by 4 (6 is not divisible by 4). ✓
Pair (6,12): z with 6|z|12 and 6<z<12: none in D₁₂. ✓
Pair (4,6): 4 and 6 are incomparable (4∤6 and 6∤4), so the condition x≤z≤y is vacuous. ✓
Convex. ✓

{1,2,6}: NOT convex (missing 3). {4,6,12}: convex.

Q8What is Pr(N = 2) for the paint problem — the probability you need exactly 2 pushes?▼
Solution
Pr(N = 2) = Pr(N > 1) − Pr(N > 2).
Pr(N > 1) = 1/1! = 1. After 1 push, U₁ < 1 always (Uniform[0,1] is strictly less than 1 with probability 1). So you always need at least 2 pushes.
Pr(N > 2) = 1/2! = 1/2. After 2 pushes, U₁+U₂ < 1 with probability 1/2 (the 2-simplex has volume 1/2).
Pr(N = 2) = 1 − 1/2 = 1/2.

Alternatively: Pr(N=2) = Pr(U₁ < 1 and U₁+U₂ ≥ 1) = Pr(U₂ ≥ 1−U₁). Integrating: ∫₀¹ Pr(U₂ ≥ 1−u) du = ∫₀¹ u du = 1/2. ✓

Pr(N = 2) = 1/2. There's a 50% chance exactly 2 pushes are needed.

Graded Posets
  • 3 conditions: (i) rk(minimal)=0; (ii) covers +1; (iii) all maximal elements same rank h(P)−1
  • Condition (iii) is essential — prevents disjoint chains of different lengths
  • Equivalent: all maximal chains same length

  • Test B: assign ranks, detect conflicts
  • Each rank level is an antichain
  • Bₙ: rk(S)=|S|. Dₙ: rk(d)=Ω(d). Chain: rk(i)=i−1
Mirsky Theorem & |P| ≤ h·w
  • Aₖ = {x : longest chain ending at x has length k}
  • Aₖ is always an antichain
  • h(P) antichains, each size ≤ w(P)
  • Therefore: |P| ≤ h(P)·w(P)
Erdős–Szekeres
  • Statement: rs+1 distinct reals ⟹ inc subseq of length r+1 OR dec of length s+1
  • Proof: pigeonhole on (uᵢ,dᵢ) pairs — all distinct
  • Special case r=s=n: n²+1 reals → monotone subseq of length n+1
  • Equivalent to |P| ≤ h·w on the sequence-poset
  • Tight example: [3,2,1,6,5,4,9,8,7] — length 9, no mono subseq of length 4
